## 1. 화면 개요 / 목적

### 1.1 화면 정보
- 화면명: 대시보드 생성 진입 화면
- 주소: home.html (접속 시 index.html에서 자동 이동)
- 브라우저 탭 제목: 설문조사 분석 대시보드 만들기 | purple6studio

### 1.2 목적
1. 설문조사 분석 대시보드를 만들기 위해 설문 제목을 입력받는다.
2. 분석에 필요한 3종 CSV 파일을 업로드받는다.
   - **문항 코드북** (=설문의 설계도): 설문이 어떻게 구성되어 있는지를 한 줄씩 정리한 파일. 한 행 당 1개의 문항 정보(문항 번호, 카테고리, 문항명, 응답 형식 등)가 담기며, 대시보드가 각 응답값을 올바르게 읽고 집계·시각화하는 기준표 역할을 한다.
   - **응답 데이터셋_숫자형** (=설문의 결과, VALUE): 각 문항에 대해 응답자가 남긴 답변을 숫자 코드로 기록한 파일. 한 행 당 1명의 응답자 정보가 담기며, 계산·집계·필터링·차트 생성 등 분석에 주로 사용한다.
   - **응답 데이터셋_텍스트형** (=설문의 결과, LABEL): 응답 내용을 읽기 쉬운 텍스트로 기록한 파일. 결과 검토, 화면에 텍스트를 자연스럽게 표시할 때 활용한다.
3. 업로드된 파일의 형식, 필수 컬럼, 상호 정합성을 검증한다.
4. 저장된 대시보드 리스트를 조회하고 다시 열기, 이름 변경, 삭제를 할 수 있는 진입점을 제공한다.
5. 검증을 통과하면 설문 데이터를 저장한 뒤 대시보드 화면으로 이동한다. *(저장 방식 TBD)*
6. 파일 준비가 되지 않은 사용자를 위해 가이드 문서로 이동할 수 있는 진입점을 제공한다.

### 1.3 사용자 여정
1. 화면 진입 후 저장된 대시보드 리스트 버튼으로 기존 대시보드를 확인한다.
2. 제목 입력 → 3개 CSV 업로드 → 대시보드 만들기 클릭
3. 검증 실패 시 인라인 오류 표시, 통과 시 대시보드 화면으로 이동

## 2. 레이아웃 구조 / 영역 구분

### 2.1 전체 레이아웃
- 흰 배경, 세로 단일 컬럼, 가운데 정렬
- 본문 최대 너비 840px

### 2.2 페이지 구조 (위에서 아래)

```text
상단 유틸 바      로고 + 저장된 대시보드 리스트 버튼
헤더              페이지 제목
본문
  섹션 1          설문조사 제목 입력
  섹션 2          파일 업로드 (3종)
  액션 버튼       대시보드 만들기
  가이드 배너     가이드 문서 링크
푸터
모달              저장된 대시보드 리스트 (기본 숨김)
```

**각 요소의 기능**
- 상단 유틸 바: 홈으로 이동하거나 저장된 대시보드 리스트를 여는 영역
- 헤더: 현재 화면이 설문조사 분석 대시보드를 만드는 화면임을 보여주는 영역
- 섹션 1: 분석할 설문조사 제목을 입력하는 영역
- 섹션 2: 코드북, 숫자형 응답 데이터, 텍스트형 응답 데이터를 각각 업로드하고 오류를 확인하는 영역
- 액션 버튼: 모든 입력과 검증이 끝난 뒤 대시보드 생성을 시작하는 버튼
- 가이드 배너: 파일 준비 방법 가이드를 여는 링크 영역
- 푸터: 저작권 정보를 표시하는 영역
- 모달: 저장된 대시보드를 다시 열거나 이름 변경, 삭제를 할 수 있는 영역

### 2.3 반응형 동작
- 화면 너비가 좁아지면 상단 유틸 바가 줄바꿈될 수 있다.
- 페이지 제목 크기와 업로드 카드 높이는 화면 폭에 맞춰 자동 조정된다.
- 모바일에서는 3열 업로드 카드가 세로로 쌓인다.

### 3.2 헤더
- 텍스트: 설문조사 분석 대시보드 만들기
- 가운데 정렬, 페이지 대표 제목 역할

### 3.3 본문

#### 섹션 1 — 설문조사 제목 입력
- 섹션 제목: 1. 설문조사 제목을 입력해 주세요
- 텍스트 입력 필드
- placeholder: 예: 2026 직장인 사무환경 조사
- 최대 50자
- 보조 문구: 최대 50자까지 입력할 수 있습니다.
- 오류 문구: 설문조사 제목을 입력해 주세요.

#### 섹션 2 — 파일 업로드
- 섹션 제목: 2. 문항 코드북과 응답 데이터셋을 업로드해 주세요
- 3열 그리드로 업로드 카드 3장 배치

**업로드 카드 (3장 공통)**

| 항목 | 내용 |
| --- | --- |
| 카드 라벨 | 문항 코드북 / 응답 데이터셋_숫자형 / 응답 데이터셋_텍스트형 |
| 허용 파일 형식 | CSV만 지원 (.xlsx 미지원) |
| 안내 문구 | 여기에 파일을 드래그하거나, 아래 버튼을 눌러 파일을 선택하세요. |
| 형식 표기 | ※ CSV 파일만 지원합니다 |
| 파일 선택 버튼 | 검정 배경, 흰 글자 |

**카드별 파일 검증 로직 (파일 업로드 시 즉시 실행 — 1차)**

- 문항 코드북 카드
  - 파일이 비어 있으면 오류 처리
  - 첫 행 헤더에 `question_no`, `question_label`, `response_type`, `data_column_role`, `value_code_map` 5개 컬럼이 모두 있어야 함
  - 하나라도 없으면 `문항 코드북 형식이 올바르지 않습니다. 누락된 컬럼: ...` 오류 표시

- 응답 데이터셋_숫자형 카드
  - 파일이 비어 있으면 오류 처리
  - 첫 번째 열이 `survey_year`, 두 번째 열이 `respondent_no`이어야 함 (컬럼명 위치 검증)
  - 첫 번째 데이터 행의 `survey_year` 값이 4자리 숫자 연도(예: 2024)이어야 함 (값 패턴 검증)
  - 위 조건 중 하나라도 실패하면 `응답 데이터셋 형식이 올바르지 않습니다. ...` 오류 표시

- 응답 데이터셋_텍스트형 카드
  - 파일이 비어 있으면 오류 처리
  - 첫 번째 열이 `survey_year`, 두 번째 열이 `respondent_no`이어야 함 (컬럼명 위치 검증)
  - 첫 번째 데이터 행의 `survey_year` 값이 4자리 숫자 연도(예: 2024)이어야 함 (값 패턴 검증)
  - 위 조건 중 하나라도 실패하면 `응답 데이터셋 형식이 올바르지 않습니다. ...` 오류 표시

**카드 하단 개별 오류 메시지**

- 지원하지 않는 파일 형식입니다. .csv 파일만 업로드할 수 있습니다.
- 파일을 읽는 중 오류가 발생했습니다.
- 문항 코드북 형식이 올바르지 않습니다.
  누락된 컬럼: {목록}
- 응답 데이터셋 형식이 올바르지 않습니다.
  첫 번째 열이 survey_year이어야 합니다.
- 응답 데이터셋 형식이 올바르지 않습니다.
  두 번째 열이 respondent_no이어야 합니다.
- 응답 데이터셋 형식이 올바르지 않습니다.
  survey_year 값이 올바른 연도 형식(예: 2024)이 아닙니다.
- 파일이 비어 있습니다.

#### 대시보드 만들기 버튼
- 활성 조건: 제목 입력 + 3종 파일 모두 업로드 완료
- 활성 상태: 검정 배경, 흰 글자
- 비활성 상태: 회색 배경, 클릭 불가
- 클릭 시 동작 : 3종 파일 정합성 검증 → 통과 시 저장 후 대시보드 이동, 실패 시 공통 오류 표시

**3종 파일 간 정합성 검증 (대시보드 만들기 버튼 클릭 시 실행 — 2차)**

- 코드북 기반 숫자형/텍스트형 판별: 코드북의 `value_code_map`이 있는 객관식 단일/중복/순위 raw 문항을 찾아, 숫자형 카드의 해당 컬럼 샘플값이 텍스트면 오류, 텍스트형 카드의 해당 컬럼 샘플값이 코드이면 오류 (판별 가능한 문항이 없으면 통과)
- 코드북 ↔ 숫자형: 코드북에 적힌 문항 이름과 순서가 숫자형 데이터 파일의 문항 컬럼(3번째 컬럼부터)과 같은지 확인
- 코드북 ↔ 텍스트형: 코드북에 적힌 문항 이름과 순서가 텍스트형 데이터 파일의 문항 컬럼(3번째 컬럼부터)과 같은지 확인
- 숫자형 ↔ 텍스트형: 두 파일의 헤더 구조, 전체 행 수, 각 행의 앞 2개 기준 컬럼 값, 실제 응답이 들어 있는 칸의 위치가 같은지 확인
- 실패 시 공통 오류 메시지 표시
- 통과 시 데이터 저장 후 대시보드 화면으로 이동 *(저장 방식 TBD)*

**공통 오류 메시지**

- 문항 코드북과 응답 데이터셋_숫자형의 문항 수가 다릅니다.
  코드북 {N}개, 데이터셋 {N}개입니다.
- 문항 코드북과 응답 데이터셋_숫자형의 문항 순서 또는 이름이 다릅니다.
  {N}번째 응답 데이터 컬럼을 확인해 주세요.
- 문항 코드북과 응답 데이터셋_텍스트형의 문항 수가 다릅니다.
  코드북 {N}개, 데이터셋 {N}개입니다.
- 문항 코드북과 응답 데이터셋_텍스트형의 문항 순서 또는 이름이 다릅니다.
  {N}번째 응답 데이터 컬럼을 확인해 주세요.
- 응답 데이터셋 숫자형과 텍스트형의 가로 첫행 구조가 서로 다릅니다.
- 응답 데이터셋 숫자형과 텍스트형의 행 수가 다릅니다.
  숫자형 {N}행, 텍스트형 {N}행입니다.
- 응답 데이터셋 숫자형과 텍스트형의 세로 첫행 기준값이 {N}번째 행에서 다릅니다.
- 응답 데이터셋 숫자형과 텍스트형의 값 위치 구조가 다릅니다.
  {N}번째 행 / {컬럼명} 컬럼을 확인해 주세요.
- 응답 데이터셋_숫자형 카드에 텍스트형 파일이 업로드된 것 같습니다.
  숫자 코드가 담긴 숫자형 파일을 업로드해 주세요.
- 응답 데이터셋_텍스트형 카드에 숫자형 파일이 업로드된 것 같습니다.
  텍스트 응답이 담긴 텍스트형 파일을 업로드해 주세요.

### 3.5 저장된 대시보드 리스트 모달
- 기본 상태: 숨김 (display none)
- 트리거: 상단 리스트 버튼 클릭
- 닫기: 닫기 버튼 / 배경 클릭 / Esc

**빈 상태**
- 메시지: 아직 저장된 대시보드가 없습니다. 설문조사 파일을 업로드하고 대시보드를 만들면 여기에 쌓입니다.

**리스트 아이템**
- 좌측: 설문 제목 + 저장 날짜 (클릭 시 해당 대시보드 열기)
- 우측: 이름 변경 버튼 / 삭제 버튼
- 이름 변경 시 인라인 입력 필드로 전환, Enter 저장 / Esc 취소 / 포커스 아웃 시 자동 저장
- 삭제 시 확인 후 제거
