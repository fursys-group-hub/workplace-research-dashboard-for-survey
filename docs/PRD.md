# PRD — 설문조사 분석 대시보드
최종 갱신: 2026-05-11
작성: purple6studio / (주)퍼시스

---

## 1. 제품 개요

### 무엇을 만드는가
설문조사 데이터를 CSV 파일로 업로드하면, 문항별 차트와 그룹 비교 분석을 즉시 볼 수 있는 웹 대시보드를 자동으로 생성한다.

### 왜 만드는가
설문 결과를 Excel이나 Tableau로 시각화하려면 기능을 익히는 데 시간이 걸리고, 그 과정에서 정작 중요한 분석에 집중하기 어렵다.
이 서비스는 데이터 파일만 올리면 문항별 최적의 시각화를 바로 확인할 수 있다. 도구를 배우는 시간 없이, 분석에만 집중할 수 있다.

### 누가 쓰는가
설문 데이터를 분석하고 결과를 보고해야 하는 내부 담당자. 데이터 분석 전문가가 아니어도 쓸 수 있도록 설계한다.

## 2. 핵심 개념

이 제품을 이해하려면 세 가지 파일 개념을 알아야 한다.

| 파일 | 역할 | 한 행의 의미 | 값의 형태 |
|------|------|-------------|----------|
| **문항 코드북** | 설문의 설계도. 각 문항의 번호·카테고리·문항명·응답 형식·보기 코드 등을 정의한다. 대시보드는 이 파일을 기준으로 데이터를 해석한다. | 1행 당 문항 1개의 정보 | — |
| **응답 데이터셋_숫자형** | 집계·계산·차트 생성에 사용하는 응답 데이터. 객관식 보기는 숫자 코드로, 척도는 정수로 저장된다. | 1행 당 응답자 1명의 전체 응답 | `1`, `2`, `3` … |
| **응답 데이터셋_텍스트형** | 화면에 텍스트로 표시할 때 사용하는 응답 데이터. 숫자형과 행 수·열 순서가 완전히 동일하며, 코드 대신 실제 보기 텍스트가 들어 있다. | 1행 당 응답자 1명의 전체 응답 | `"남"`, `"매우 그렇다"` … |

세 파일은 항상 같은 설문의 것이어야 하며, 문항 구조(열 순서)와 응답자 수(행 수)가 일치해야 한다. 숫자형과 텍스트형은 같은 응답자의 같은 응답을 서로 다른 형태로 표현한 쌍이다.

## 3. 화면 구성

### 3.0 로그인 화면 (`login.html`)

서비스 진입 전 비밀번호 관문.

Supabase 인증 기반의 단일 비밀번호 게이트로, 세션은 24시간 유지된다. 세션이 없거나 만료되면 모든 화면에서 자동으로 이 화면으로 돌아온다. 사용자별 데이터 분리는 하지 않으며(공용 작업 공간), 외부 노출을 막는 용도다.

### 3.1 홈 화면 (`home.html`)

대시보드를 새로 만들거나 기존 대시보드를 여는 진입점.

사용자는 이 화면에서 분석할 설문의 제목을 입력하고, 3종 CSV 파일(문항 코드북, 응답 데이터셋_숫자형, 응답 데이터셋_텍스트형)을 업로드한다. 파일을 올리면 형식과 정합성을 자동으로 검증하고, 문제가 없으면 대시보드 화면으로 이동한다.

이전에 만든 대시보드가 있으면 리스트에서 바로 다시 열 수 있고, 이름 변경이나 삭제도 여기서 한다.

**주요 기능**
- 설문 제목 입력 (최대 50자)
- 3종 CSV 파일 업로드 (드래그&드롭 또는 파일 선택)
- 파일 검증 (형식, 필수 컬럼, 3종 파일 간 정합성) → 상세 규칙은 `docs/ui_spec_home.md` 참조
- 저장된 대시보드 리스트 조회 / 이름 변경 / 삭제

**흐름**
```
제목 입력 → 3종 파일 업로드 → 검증 통과 → 대시보드 화면으로 이동
```

### 3.2 대시보드 화면 (`dashboard.html`)

실제 분석을 수행하는 메인 화면.

화면 왼쪽에는 코드북 기반 문항 목록이 트리 형태로 펼쳐진다. 사용자는 보고 싶은 문항을 드래그해 오른쪽 드롭존에 올려놓는다. 문항을 올려놓으면 해당 문항의 응답 유형에 맞는 차트와 데이터 테이블이 자동으로 생성된다.

그룹 비교 드롭존에 기준 문항(예: 성별, 세대)을 올려놓으면, 모든 결과가 그룹별로 나뉘어 비교된다. 필터 바에서는 분석 대상 응답자를 조건으로 좁힐 수 있다.

결과 차트는 이미지로 저장할 수 있어 보고서 작성에 바로 활용할 수 있다.

**주요 기능**
- 상단 헤더: 새 대시보드 만들기 버튼 (홈 화면으로 이동), 저장된 대시보드 리스트 버튼, 데이터 교체하기 버튼
- 좌측 문항 패널: 코드북 기반 문항 트리 (카테고리별 아코디언, 검색)
- 필터 바: 응답자 모집단 필터링 (N 카운트 실시간 갱신)
- 분석 구조 정의: 문항을 드래그해 두 드롭존에 배치
  - 보고 싶은 문항 (최대 20개)
  - 그룹별 비교 (최대 1개)
- 연도별 비교: 응답 데이터의 `survey_year` 열을 기준 문항으로 자동 사용
- 여러 척도 문항 한 번에 비교: 같은 척도 길이의 척도 문항들을 묶어서 비교
- 결과 영역: 문항 유형에 맞는 차트 + 데이터 테이블
- 차트 이미지 추출: 각 결과 섹션을 PNG로 저장 (섹션 7 참고)
- 데이터 교체하기: CSV 파일 교체 및 재검증
- 설문 제목 인라인 수정

## 4. 지원 문항 유형과 시각화

| 문항 유형 | 기본 차트 | 추가 차트 옵션 | 그룹 비교 | 여러 문항 비교 |
|----------|---------|--------------|---------|--------------|
| 객관식 단일 | 가로 막대 | 세로 막대 / 가로 100% 누적 막대 / 원형(파이) | ✓ | — |
| 객관식 중복 | 가로 막대 | 세로 막대 | ✓ | — |
| 객관식 순위 | 가로 롤리팝 | 세로 롤리팝 / 가로 누적 막대 / 세로 누적 막대 | ✓ | — |
| 객관식 척도 (원문항) | 가로 100% 누적 막대 | 척도 평균 그래프 / 원형(파이) | ✓ | ✓ (같은 척도 길이) |
| 객관식 척도 (원문항 활용 산출값) | 박스플롯 | 척도 평균 그래프 | ✓ | ✓ (같은 척도 길이) |
| 주관식 숫자 | 박스플롯 | 히스토그램 | ✓ | — |
| 주관식 시간 | 박스플롯 | 히스토그램 | ✓ | — |
| 주관식 비율 배분 | 가로 100% 누적 막대 | 원형(파이) | ✓ | — |
| 주관식 문자 | 응답 목록 (키워드 검색) | — | — | — |

- 주관식 시간은 코드북의 `derived` 분환산 열을 자동으로 사용해 주관식 숫자와 동일한 방식으로 분석되며, 시간 포맷(`hh:mm`)으로 표시된다.
- 주관식 문자는 모든 응답을 표시하고 키워드로 즉시 필터링할 수 있다.

## 5. 데이터 저장 방식

파일과 설문 메타 정보는 Supabase에 저장한다.

| 저장소 | 저장 내용 |
|--------|---------|
| Supabase DB (`surveys` 테이블) | 설문 메타 정보 (제목, 생성일, 공유 토큰 등) |
| Supabase DB (`survey_files` 테이블) | 파일 역할(codebook / value / label), 파일명, 스토리지 경로 |
| Supabase Storage (`survey-files` 버킷) | 실제 CSV 파일 (`shared/{surveyId}/{role}.csv`) |
| sessionStorage | 화면 간 임시 데이터 (현재 설문 ID, 제목) |

- 파일은 `shared/{surveyId}/codebook.csv` 형태의 경로로 Storage에 업로드한다.
- 설문 삭제 시 DB 레코드와 Storage 파일을 함께 제거한다.

## 7. 이미지 추출

차트를 고객사 제출용 이미지로 저장할 수 있다.

- 출력 사양: PNG, 흰 배경
- 각 결과 섹션마다 "이미지로 저장" 버튼 제공
- 내부 UI 컨트롤(그래프 유형 선택, 정렬 버튼 등)은 이미지에서 제외

## 8. 현재 구현 상태

### 완료
- 로그인 화면 (Supabase 인증 기반 비밀번호 관문, 24시간 세션)
- 홈 화면 UI 및 파일 검증 로직
- 홈 화면 UI 스펙 문서 (`docs/ui_spec_home.md`)
- 대시보드 화면 UI 스펙 문서 (`docs/ui_spec_dashboard.md`)
- 대시보드 화면 전체 레이아웃 (헤더, 좌측 패널, 필터 바, 드롭존, 결과 영역)
- 문항 트리 렌더링 및 드래그앤드롭
- 필터 기능 (응답자 모집단 필터링, N 카운트)
- 결과 영역 차트 렌더링 (객관식 단일·중복·순위·척도, 주관식 숫자·시간·비율 배분·문자)
- 객관식 순위 그룹 비교
- 여러 척도 문항 한 번에 비교하기
- 연도별 비교 (`survey_year` 자동 기준)
- 사용자 정의 그룹 설정 (그룹 묶음 보기)
- 기타 응답 모달 / 텍스트 응답 검색
- 데이터 교체하기 (CSV 파일 교체 및 재검증)
- 설문 제목 인라인 수정
- 차트 이미지 추출 기능 (A4, 300DPI, PNG)
- 이미지 추출 명세 문서 (`docs/image-export-spec.md`)
- Supabase 연동 (DB 및 파일 스토리지)
- 디자인 시스템 (타이포그래피, 색상, 공통 컴포넌트)
- 문서 체계 (데이터 파이프라인, 시각화 규칙)
