# 데이터 시각화 디자인 시스템

> **범위**: 시각화 영역의 디자인 토큰, 컴포넌트 스펙, 렌더링 규칙, 데이터 집계·계산·예외 처리 규칙을 함께 다룬다.
> **연계**: 공통 디자인 토큰(폰트, 중립 색상, 반경, 그림자)은 `design_system.md`를 참조한다. 이 문서에는 시각화 영역 전용 적용 기준만 기술한다.
> **AI 작업 규칙**: 이 문서에 정의된 토큰과 컴포넌트만 사용한다. `design_system.md`와 이 문서에 없는 색상·크기·간격을 임의로 추가하지 않는다. 새 토큰이 필요하면 반드시 사용자에게 확인을 요청한다.

## 0. 기본 개념

### 분석 대상 문항
- 결과를 직접 시각화해서 보여주는 중심 문항이다.
- 결과 카드 제목은 기본적으로 코드북의 `question_label`을 사용한다.
- 질문 원문이 있으면 `Q. {question_full}` 형식으로 함께 표시한다.

### 분석 기준 문항
- 분석 대상 문항의 결과를 그룹별로 나누어 비교할 때 사용하는 문항이다.
- 현재 대시보드는 단일응답 계열 그룹 문항과 시스템 컬럼 `survey_year`를 기준 문항 후보로 사용한다.
- 기준 문항을 선택하지 않으면 전체 결과만 보여준다.

### 지원 문항 유형
- 객관식 단일
- 객관식 중복
- 객관식 순위
- 객관식 척도
- 주관식 숫자
- 주관식 시간
- 주관식 비율 배분
- 주관식 문자

### 비지원 문항 유형
- 위 지원 목록에 포함되지 않는 유형
- 비지원·집계 불가 시 시각화 영역에 가운데 정렬된 안내 블록(`neutral-50` 배경)으로 `아직 지원하지 않는 문항 형식`을 표시한다. 글꼴은 **1-1** `비지원 안내 문구` 토큰을 따른다.

### 1-2. 컬러 팔레트 용도

#### 단일 색상 그래프
- 단일 색상을 사용하는 경우(가로/세로 막대 기본 차트, 가로/세로 롤리팝 기본 차트, 히스토그램 등) `neutral-700`을 기본으로 한다.

#### 여러 색상이 필요한 그래프
- **그룹 비교**: 그룹 순서대로 `color-1` → `color-2` → … → `color-20` 순환 할당. 최대 20개 그룹까지 색상 구분 가능.
- **그룹 비교 — 겹침 모드**(가로 막대·가로 롤리팝): 응답자 전체 기준 막대·줄기는 `neutral-300`.
- **객관식 단일 — 100% 기준 누적 막대 / 원형 그래프**: 보기 순서대로 `color-1` → `color-2` → … 순환 할당해 보기별 색상을 구분.

#### 순위 계열
- 누적 막대 차트에서 1/2/3순위 구간을 색상으로 구분할 때 사용한다. neutral 계열만 사용하되 명도로 순위를 구분한다.

| 순위 | 토큰 |
|------|------|
| 1순위 | `neutral-700` |
| 2순위 | `neutral-400` |
| 3순위 | `neutral-300` |
| 4순위 | `neutral-200` |
| 4순위 | `neutral-100` |

#### 척도 계열
- 1점(최저)은 `color-2`(붉은 계열), 중간점은 `neutral-300`, 최고점은 `color-1`(푸른 계열)을 기준으로 하고, 그 사이는 자연스러운 그라데이션 값으로 채운다.
- 5점 척도와 7점 척도는 아래 가이드를 따르고, 그 외 척도는 같은 방식으로 적당히 지정한다.

**5점 척도**

| 점수 | 토큰 |
|------|------|
| 1 | `color-2` |
| 2 | `low-3` |
| 3 | `neutral-300` |
| 4 | `high-3` |
| 5 | `color-1` |

**7점 척도**

| 점수 | 토큰 |
|------|------|
| 1 | `color-2` |
| 2 | `low-4` |
| 3 | `low-2` |
| 4 | `neutral-300` |
| 5 | `high-2` |
| 6 | `high-4` |
| 7 | `color-1` |

**그 외 척도**: 위 기준점(`color-2` → `neutral-300` → `color-1`)을 유지하면서 단계 수에 맞게 중간 토큰을 적절히 배정한다.

#### 박스플롯 계열
- 박스 내부 구간(Q1–Q2, Q2–Q3)은 두 구간을 같은 단일 색상으로 채운다.
- 단일 보기(응답자 전체)는 `neutral-400`을 사용한다.
- 그룹 비교는 그룹 비교 팔레트(`color-1` → `color-2` → … 순환)를 그대로 따른다.

#### 척도 평균 그래프 계열
- 분포 막대 배경의 각 구간 색상은 척도 분포 그래프와 동일한 색상을 사용하되, opacity 0.3을 적용한다.

### 2-3. 차트

#### 2-3-0. 공통

> **방향 정의**: 막대가 가로로 뻗는 차트를 **가로 막대**, 막대가 세로로 뻗는 차트를 **세로 막대**라 한다. 가로 막대에서 보기명은 왼쪽, 비율은 오른쪽에에 표시된된다. 세로 막대에서 보기명은 하단, 비율은 위에 표시된다.

- 막대 border-radius: `--radius-4` (4px). 방향별 적용 위치는 아래와 같다.
  - **가로 막대·가로 누적 막대**: 우측 끝(우상·우하)에만 적용.
  - **가로 100% 누적 막대**: 첫 번째 구간 좌끝(좌상·좌하)과 마지막 구간 우끝(우상·우하) 적용.
  - **세로 막대·세로 누적 막대**: 상단(좌상·우상)에만 적용. 누적 막대는 최상단 구간에만.
- 수치 레이블: `Label-1-strong` (→ 1-1 타이포그래피 참조). 차트 요소가 너무 좁을 때는 표시하지 않음.
- 보기명: `Label-1` (→ 1-1 타이포그래피 참조), `neutral-900`
- 막대 그래프 (1~5번, 11번 히스토그램) 공통 축 스타일: 축 색상 `neutral-300` / 눈금 색상 `neutral-200`
- 막대 hover: opacity 0.85
- 막대 전환 애니메이션: `width 0.2s ease` (가로 막대) / `height 0.2s ease` (세로 막대)

#### 2-3-1. 가로 막대

#### 2-3-12. 그룹별 비교

기준 문항으로 응답을 나눈 뒤, 그룹마다 값이 어떻게 다른지 한 화면에서 비교할 때 쓴다. 같은 차트 안에 **겹쳐** 보여 주는 방식과, **행을 세로로 쌓아** 나열하는 방식이 있다. 아래에 나온 차트만 그룹 비교를 지원한다.

**지원 차트**

| 차트 | 방식 |
|------|------|
| 가로 막대 | 겹침(도트) / 그룹 2개일 때 이중 막대 |
| 롤리팝(가로) | 겹침(도트) / 그룹 2개일 때 이중 롤리팝 |
| 100% 기준 누적 가로 막대 | 세로 나열(lane-group) |
| 척도 평균 그래프 | 세로 나열(lane-group) |
| 박스 플롯 | 세로 나열(lane-group) |

---

**(1) 가로 막대·롤리팝(가로) — 도트로 겹쳐 보기**

응답자 전체 기준 막대(또는 롤리팝)를 `neutral-300`으로 깔아 두고, 그 위에 그룹별 값을 **지름 16px** 도트로 겹친다. 도트 색은 범례의 그룹 색과 같고, 테두리는 흰색이다. 도트를 누르면 그 그룹만 남기고 나머지는 흐리게 하며, 해당 그룹의 수치 레이블을 도트에 붙인다. 수치는 도트 **오른쪽**이 기본이나, 막대가 길어 오른쪽 여백이 부족하면 잘리지 않도록 도트 **왼쪽**에 둔다.

막대·축·수직 눈금선 등 겹침 모드의 골격은 일반 `#### 2-3-1. 가로 막대` 또는 `#### 2-3-6. 롤리팝 (가로)`와 같다.

---

**(2) 그룹이 둘일 때 — 이중 막대·이중 롤리팝**

범례에서 표시 중인 그룹이 **두 개만** 남으면, 범례 패널에 `두 그룹만 비교하기` 버튼이 노출된다. 이걸 쓰면 배경용 `neutral-300` 막대(줄기) 차트는 숨기고, **보기(행)마다** 막대 또는 롤리팝 트랙을 **세로로 두 줄** 쌓아 두 그룹을 나란히 본다. 이중 막대·이중 롤리팝 모드가 켜지면 같은 버튼 문구가 `기본 그래프로 보기`로 바뀌며, 누르면 **(1) 겹침(도트)** 모드로 되돌린다(`2-4` 범례 액션 버튼).

- 한 보기 안에서 두 막대(또는 두 롤리팝 트랙) 사이: **4px**
- 보기와 보기(행) 사이: **16px**
- 막대 쪽 축·눈금·수치 레이블 규칙은 `#### 2-3-1. 가로 막대`와 동일하다.
- 막대 채움색과 롤리팝 줄기·도트(헤드) 색은 모두 범례의 그룹 색과 같다.

---

**(3) 100% 누적·척도 평균·박스 — lane-group으로 세로 나열**

응답자 전체와 각 그룹을 **위에서 아래로** 다른 행에 그린다. 공통 레이아웃 클래스는 `.lane-group-chart`이며, 다음을 공유한다.

- **순서**: 맨 위는 항상 `응답자 전체` 한 행 → 가로 구분선(`lane-group-divider`, 좌측은 라벨 열 폭만큼 비움) → 그 아래로 그룹별 행.
- **행 사이**: 컨테이너 세로 `gap` **16px**. 구분선에는 위·아래 **8px** 마진을 더 둔다.
- **세로 치수**: **40px**. 누적 막대 트랙(`stack100-track`), 척도 분포/평균 전용 행, 박스 행, 평균 원 지름이 이 토큰에 맞춰진다.

**100% 기준 누적 가로 막대**  
막대 아래에 붙이던 보기명 레이블은 그룹 모드에서는 두지 않고, 보기 이름은 범례에서만 본다. 구간 색·비율 표기 등 나머지 골격은 `#### 2-3-3. 가로 막대 100% 기준 누적형`을 따른다.

**척도 평균 그래프**  
분포·평균 lane-group 모두 행마다 하단 보기 문구·끝점 레이블·`평균` 문구는 두지 않는다. `평균 보기`만 차트 맨 아래 공통 축에 척도점 숫자(1~N)를 둔다.

**박스 플롯**  
박스 행 높이와 평균 원은 위와 같이 40px 토큰으로 맞춘다. 평균 쪽 레이블은 척도와 같이 줄이고, `Q1`·`Q2`·`Q3` 문구는 표시하지 않는다.

---

**그룹 비교를 쓰지 않는 차트**

세로 막대, 가로·세로 막대 누적형, 롤리팝(세로), 원형(파이), 히스토그램은 그룹별 보기를 제공하지 않는다.


### 2-4. 범례 패널

#### 2-4-1. 기본 기능

차트 우측 열에 배치되는 범례 패널이다.  

##### 2-4-1-1. 범례 레이아웃

`.result-visual`은 두 열 그리드(`grid-template-columns: minmax(0, 1fr) var(--viz-legend-width)`)로 차트와 범례를 나란히 두며, 차트 열이 남은 가로 공간을 차지하고 범례 열 너비는 `--viz-legend-width` 토큰에 따른다(위 **`2-1. 결과 카드` > 시각화 영역** 표의 범례 패널 너비와 동일).

범례의 기본 구성은 `체크박스 + 색상 swatch + 항목명`이다.  
체크박스는 단순히 현재 색상과 항목을 보여주는 장식이 아니라, 해당 항목을 차트와 표에서 표시할지 숨길지를 사용자가 직접 제어하는 토글이다.  
경우에 따라 체크박스나 일괄 액션을 두지 않는 범례도 있다(아래 **범례의 종류** 및 **`## 4. 문항 응답 형식별 시각화 규칙`** 참고).

체크박스는 `design_system.md`의 Checkbox 컴포넌트 스펙을 따른다.

##### 2-4-1-2. 범례의 유형

범례 **유형**마다 색을 대응시키는 대상·체크박스 유무·액션 버튼을 아래에 정리한다.
어떤 문항·차트에서 어떤 범례가 붙는지는 **`## 4. 문항 응답 형식별 시각화 규칙`**을 참고한다. 

| 범례 종류 | 색이 붙는 대상 | 체크박스 | 액션 버튼 |
|-----------|----------------|----------|-----------|
| **그룹 범례** | 비교 집단(그룹)마다 하나의 swatch | 있음 — 그룹 행마다, 차트·표에 그 그룹을 넣을지 고름 | `전체 선택`, `전체 해제`, `그룹 편집`, `두 그룹만 비교하기`※ |
| **순위 범례** | 1순위·2순위 등 순위마다 하나의 swatch | 있음 — 순위마다, 그 순위 응답을 차트·표에 넣을지 고름 | `전체 선택`, `전체 해제` |
| **보기 범례** | 응답 보기(선택지)마다 하나의 swatch | 없음 | 없음 |
| **점수 범례** | 척도 점수(및 코드북에 연결된 그 점수의 보기 문구)마다 하나의 swatch | 없음 | 없음 |

**보기 범례**와 **점수 범례**는 표시 전용으로, 클릭·호버 등 인터랙션이 없다.

- ※ **`두 그룹만 비교하기` / `기본 그래프로 보기`**: **가로 막대**·**가로 롤리팝(순위)** 그룹 비교 모드, **척도형 여러 문항 비교**의 그룹 비교 모드에서, 범례에 남겨 둔(체크된) 그룹이 **정확히 둘**일 때만 버튼이 노출된다. 기본 문구는 `두 그룹만 비교하기`이고, 이중 막대·이중 롤리팝 모드가 켜지면 `기본 그래프로 보기`로 바뀐다. 100% 기준 누적 가로 막대·척도 평균 그래프·박스 플롯 등 lane-group 방식 그룹 비교에는 두지 않는다(`#### 2-3-12. 그룹별 비교` **(2)** 참고).

범례 패널은 공통적으로, 필요하면 서로 다른 종류의 범례를 둘 이상 한 패널에 둘 수 있으며, 그럴 경우 구획 사이에 구분선을 둔다.

**범례 기본 스펙**

| 항목 | 기준 |
|------|------|
| 너비 | `var(--viz-legend-width)` (수치·식은 `2-1` 시각화 영역 표와 동일) |
| 패딩 | 없음 |
| **줄 사이** 간격 | 세로로 쌓인 **범례 행**과 행 사이, `gap: 8px` |
| **한 줄 안** 간격 | **같은 행**에서 체크박스·swatch·텍스트 사이, `gap: 8px` |
| 체크박스·swatch 크기 | 14px × 14px |
| swatch border-radius | 4px |
| 범례 항목 텍스트 | `Label-1`, `neutral-800` |
| 범례 항목 추가 텍스트 | `Label-1`, `neutral-600` (예: 점수 옆 보기 문구) |
| 비활성 텍스트 | `neutral-300` |
| 비활성 swatch | opacity 0.28 |
| 항목 블록 ↔ 액션 첫 줄 | 세로 간격 **24px** |
| 액션 버튼끼리 | 가로 간격 **8px** (줄 바꿈 가능) |
| `두 그룹만 비교하기` / `기본 그래프로 보기` ↔ 위 액션 줄 | 세로 간격 **8px** |

##### 2-4-1-3. 범례 액션 버튼

**그룹 범례**·**순위 범례**에만 아래 버튼 묶음이 올 수 있다. **보기 범례**·**점수 범례**에는 액션 버튼을 두지 않는다(위 **범례의 종류** 표).

| 버튼 | 붙는 범례 유형 | 역할 |
|------|----------------|------|
| `전체 선택` | 그룹 범례, 순위 범례 | 그 범례 구획의 체크박스를 모두 켠다 |
| `전체 해제` | 그룹 범례, 순위 범례 | 모두 끈다 |
| `그룹 편집` | 그룹 범례만 | `범례 그룹 편집` 모달을 연다 |
| `두 그룹만 비교하기` / `기본 그래프로 보기` | 일부 그룹 범례만 | 위 **범례의 종류** ※. 같은 버튼이 토글된다. `두 그룹만 비교하기`: **가로 막대·가로 롤리팝(순위)**에서는 `#### 2-3-12. 그룹별 비교` **(2) 이중 막대·이중 롤리팝** 모드로, **척도형 여러 문항 비교**에서는 문항 행마다 그룹별 롤리팝 트랙을 세로로 쌓는 형태로 전환. `기본 그래프로 보기`: 이중 모드를 끄고 겹침(도트) 또는 해당 차트의 기본 그룹 비교 보기로 되돌린다. |

- `design_system.md`: `전체 선택`·`전체 해제`는 **`Button-2 Default`**, `그룹 편집`은 **`Button-2 Gray`**. `두 그룹만 비교하기`는 **`Button-2 Black`** 계열(배경 `neutral-900`). 이중 모드가 켜져 `기본 그래프로 보기`로 보일 때는 배경 `white`·텍스트 `neutral-900`·regular(`is-active`).

#### 2-4-2. 범례 그룹 편집 모달

##### 2-4-2-1. 기본 구조

- 범례에서 `그룹 편집` 버튼을 누르면 `범례 그룹 편집` 모달이 열린다.
- 사용자는 여러 범례 항목을 하나의 그룹으로 묶어 차트와 데이터 테이블에서 비교 단위를 재구성할 수 있다.
- 모달은 **헤더**·**본문**·**푸터**로 나뉜다.

**헤더**

| 항목 | 기준 |
|------|------|
| 타이틀 | `범례 그룹 편집` |
| 닫기 버튼 | 헤더 우측 `×` 아이콘. 누르면 편집 내용을 저장하지 않고 모달을 닫는다. |

**본문**

스크롤 가능한 **단일 세로 리스트**다. 분석 기준의 그룹 값을 **원본 범례 순서**대로 나열하며, 행 종류는 아래 두 가지뿐이다.

| 행 종류 | 언제 쓰이는지 | 구성 |
|---------|----------------|------|
| **미배정 항목 행** | 아직 사용자 정의 그룹에 넣지 않은 값 | 체크박스 + swatch + 항목명(위 **`2-4. 범례 패널` > 범례 기본 스펙**; 체크박스는 `design_system.md` Checkbox). 드래그하여 기존 그룹으로 옮길 수 있음 |
| **그룹 아코디언** | 하나 이상의 범례 항목을 묶어 만든 사용자 정의 그룹 | 헤더 + 본문 |

**푸터**

푸터는 액션 버튼으로 이루어진다.

| 항목 | 기준 |
|------|------|
| 위치 | 모달 하단 |
| 구성 | 좌: `+` 아이콘 + `그룹 만들기` / 우: `전체 해제`, `편집 완료` |
| 버튼 스타일 | `그룹 만들기`·`전체 해제`는 `Button-2 Default`(비활성 시 `Button-2 Inactive`). `편집 완료`는 `Button-2 Black`(비활성 시 배경만 `neutral-300`으로 연해지며, Inactive variant는 쓰지 않음) |
| `그룹 만들기` | 본문에서 선택한 미배정 항목으로 새 아코디언 그룹을 만든다. 선택된 미배정 항목이 없으면 비활성 |
| `전체 해제` | 만든 그룹을 모두 없애 모든 항목을 미배정 행으로 되돌린다. 그룹이 하나도 없으면 비활성 |
| `편집 완료` | 현재 편집 내용을 차트·표에 적용하고 모달을 닫는다. 변경이 없으면 비활성 |

아래 **편집 흐름 — …** 절은 모달 안에서 사용자가 거치는 순서를 설명한다. 

##### 2-4-2-2. 편집 흐름 — 최초 진입

- 저장해 둔 사용자 정의 그룹이 없을 때, 모든 범례 항목이 **미배정 항목 행**만 나열된다. 그룹 아코디언은 없다.
- 사용자는 미배정 항목을 체크박스로 고를 수 있다.

##### 2-4-2-2. 편집 흐름 — 그룹 만들기

1. 미배정 행에서 묶을 항목을 체크박스로 **하나 이상** 선택한다.
2. `그룹 만들기` 버튼을 누른다.
3. 선택 항목이 **아코디언 그룹**으로 바뀐다.
   - 리스트에서는 멤버 중 **원본 순서가 가장 앞선 항목** 자리에 블록이 들어간다.
   - 아코디언은 **펼침** 상태로 표시된다.
   - 헤더 구성: 접기·펼치기, 그룹 swatch, 그룹명(한 줄 입력란), **이름 수정**·**그룹 해제** 아이콘.
   - 본문 구성: 멤버별 swatch·항목명, **항목 제거** 아이콘.
   - 그룹명 입력란에 입력 커서가 이동하고 텍스트는 전체 선택 상태다.
   - 그룹 swatch 색은 `customGroupDefs` 배열의 생성 순서에 따라 사용자 정의 그룹 전용 팔레트(`var(--color-11)` ~ `var(--color-18)`, 순환)에서 자동 배정된다.
4. 사용자가 그룹명을 입력·수정한다. `Enter` 또는 입력란 바깥 클릭으로 확정하며, 비워 두고 확정하면 `그룹1`, `그룹2`, … 순으로 기본 이름이 부여된다.
5. 선택 체크는 모두 해제된다. 아직 묶지 않은 값은 **미배정 항목 행**으로 남는다. 1~5를 반복해 그룹을 여러 개 만들 수 있다.

##### 2-4-2-2. 편집 흐름 — 그룹 수정/편집

그룹을 하나라도 만들면, 본문 리스트에는 **미배정 항목 행**과 **그룹 아코디언**이 원본 범례 순서대로 섞여 보인다. 그룹에 속한 값은 미배정 행으로 다시 나오지 않고, 해당 아코디언 **본문**에만 보인다. 이 상태에서 아래처럼 고칠 수 있으며, 변경이 생기면 `편집 완료`가 활성화된다.

- **그룹 이름 바꾸기** — 확정된 그룹명은 헤더에 **텍스트**로 보인다. **이름 수정** 아이콘을 누르면 한 줄 입력란으로 바뀌고, 이름을 고친 뒤 `Enter` 또는 입력란 바깥 클릭으로 다시 확정한다. 이름을 작성하지 않고 확정하면 `그룹1`, `그룹2`, … 순으로 기본 이름을 부여한다. 서로 다른 그룹이 같은 이름을 써도 된다.
- **기존 그룹에 항목 추가** — `그룹 만들기`와 별도로, 미배정 행을 드래그해 기존 아코디언 위에 놓으면 그 그룹에 추가할 수 있다. 드래그하여 놓으면 해당 그룹은 펼쳐진다.
- **기존 그룹의 항목 제거** — 아코디언 본문의 **항목 제거** 아이콘을 누르면 그 항목만 그룹에서 빠져 **미배정 항목 행**으로 돌아간다. 그룹에 남은 멤버가 없으면 그룹은 자동으로 해제된다.
- **그룹 삭제** — 아코디언 헤더의 **그룹 해제** 아이콘을 누르면 해당 그룹은 삭제되고, 속했던 항목은 모두 **미배정 항목 행**으로 돌아간다.
- **모든 그룹 삭제** — 푸터의 **전체 해제** 버튼을 누르면 만든 그룹이 모두 없어지고, 모든 항목이 미배정 행으로 돌아간다.

##### 2-4-2-2. 편집 흐름 — 편집 완료 후 적용

- 푸터의 `편집 완료` 버튼을 누르면 편집된 그룹 구성이 적용되고 모달이 닫힌다. 모달을 연 뒤 변경을 하지 않았으면 버튼은 활성화되지 않는다.
- 적용된 그룹은 차트·데이터 테이블 양쪽에서 **새 범례 항목** 하나로 취급한다. 그룹에 넣은 원본 범례 항목은 차트·표 집계 단위로는 더 이상 각각 따로 두지 않는다. 그룹에 넣지 않은 항목은 원래 범례 항목으로 그대로 남는다.

**그룹 편집 후 대시보드 상의 범례 표시**

- **사용자 정의 그룹**은 범례 패널에서 **그룹 한 줄**로 보인다. 체크박스 + 그룹 색 swatch + 사용자가 지정한 그룹명(예: `그룹1`, `그룹2`)이며, 위 **`2-4. 범례 패널` > 범례 기본 스펙**을 따른다.
- 그룹에 묶었던 **원본 범례 항목**은 같은 그룹 줄 **아래에 들여쓴 하위 목록**으로 함께 표시한다. 각 항목은 원래 범례와 같이 swatch + 항목명(예: `세대: Z세대(1996-2001)`)만 보이고, **체크박스는 없다**. 표시·숨김은 그룹 줄의 체크박스로만 제어한다.
- 하위 항목 swatch는 원본 그룹 색을 유지하되 `opacity: 0.7`을 적용해 그룹 줄 swatch보다 연하게 구분한다. 하위 항목 텍스트는 `neutral-700`이다. 그룹을 끄면(비활성) 하위 목록 전체에 `opacity: 0.28`을 적용한다.
- 그룹에 넣지 않은 원본 범례는 기존처럼 **그룹 범례 한 줄**(체크박스 + swatch + 항목명)로만 나열된다.

##### 2-4-2-2. 편집 흐름 — 재진입

- 이전에 `편집 완료`로 저장해 둔 사용자 정의 그룹이 있는 상태에서 모달을 열면, **그룹 수정/편집**과 같은 혼합 리스트 상태로 시작한다. **최초 진입**(미배정만 있는 상태)으로 돌아가지 않는다.

##### 2-4-2-2. 편집 흐름 — 예외 처리

- 선택 항목이 없으면 그룹 생성은 불가하다.
- 동일 항목을 둘 이상의 그룹에 중복 포함할 수 없다.
- 그룹명 중복은 허용한다.
- 빈 그룹은 허용하지 않으며, 항목이 모두 제거된 경우 그룹 해제로 처리한다.

### 2-5. 데이터 테이블

문항·차트 유형별 **열 구성·행 구조·합계 행·하단 안내 문구**는 **`## 4. 문항 응답 형식별 시각화 규칙`** 각 절의 `#### 데이터 테이블`을 따른다. 이 절은 모든 표에 공통으로 적용되는 **레이아웃·타이포·보더·액션 버튼** 스타일만 정의한다.

#### 2-5-1. 배치

| 항목 | 기준 |
|------|------|
| 위치 | 차트 행 아래(→ `#### 시각화 영역`의 차트↔테이블 간격 48px) |
| 섹션 좌·우 패딩 | 16px |

#### 2-5-2. 테이블 공통

| 항목 | 기준 |
|------|------|
| 너비 | 100%, `border-collapse: collapse` |
| 셀 패딩 | 8px 16px |
| 기본 정렬 | 좌측 (`text-align: left`) |
| 줄바꿈 | `white-space: nowrap` |
| 글꼴(본문·일반 셀) | `Label-2`, `neutral-700`, line-height 1.6 |
| 글꼴(헤더·합계) | `Label-2-strong`, `neutral-700`, line-height 1.6 |
| 본문 행 구분선 | `tbody` 일반 `td` **하단** 1px solid `neutral-200` (본문 행 사이). `tbody` **마지막** 행의 `td`는 하단 보더 없음 |

수치 열(`응답 비율`, `응답 수`, 평균 등)에는 우측 정렬과 고정폭 숫자(`tabular-nums`)를 적용한다.

#### 2-5-3. 헤더 열·그룹 열 

| 항목 | 기준 |
|------|------|
| 첫 번째 열(보기·문항명 등) | 가로 스크롤 시 좌측 고정(sticky), 배경 `neutral-50` |
| 첫 번째 열과 그 다음 열 사이 | **첫 번째 열 오른쪽**에만 1px solid `neutral-300` 구분선. 바로 옆 열(헤더 2열·본문 2열 등)은 **왼쪽 보더 없음** — 이중 선이 나지 않도록 함 |
| 그룹·순위 블록 경계 | `group-col` 등 블록이 시작되는 열 **왼쪽**에 1px solid `neutral-300` (첫 번째 열 바로 다음이 아닌, 그룹·순위 구간 경계에만) |

#### 2-5-4. 헤더·본문·합계 행

| 구분 | 기준 |
|------|------|
| 헤더 행(`th`) | 배경 `neutral-50`, `Label-2-strong`. **하단** 1px solid `neutral-300` (헤더↔본문·다중 헤더 행 사이) |
| 본문 행(`tbody`의 일반 `tr`) | 배경 `white`, `Label-2`. **상·좌·우 보더 없음** — 아래 행과의 경계만 셀 **하단** 1px solid `neutral-200` (`2-5-2`). 열 사이 세로선은 `2-5-3`만 적용 |
| 합계 행 | **항상 있지는 않음** — 문항·표 유형마다 `## 4`에 따름. 있을 때: 배경 `neutral-50`, `Label-2-strong`, **상단** 1px solid `neutral-300`으로 본문과 구분,  마지막 행이므로로 하단 보더 없음 |

#### 2-5-5. 평균값 셀

척도·파생 척도 등에서 평균 수치 셀에 적용한다. 비율 배분 표는 예외로 일반 수치 셀과 동일한 굵기·색을 유지한다.

| 대상 | 글꼴 | 텍스트 색상 |
|------|------|-------------|
| 일반(`.mean-value`) | `Label-2-strong` | `neutral-900` |
| 비율 배분 표의 평균 셀 | `Label-2` | `neutral-700` |

여러 문항 척도 비교 표에서 평균 열(`mean-col`) 왼쪽에는 `group-col`과 같이 1px solid `neutral-300`을 둔다.

#### 2-5-6. 보충 설명

| 항목 | 기준 |
|------|------|
| 글꼴 | `Caption-1`, `neutral-600` |
| 위치 | 테이블 래퍼 바로 아래 |
| 테이블과의 간격 | 8px |

문구 내용은 문항 형식별로 `## 4`에 정의한다(예: 객관식 중복 비율 합계 안내).

- 글꼴: `Label-1-strong`, `neutral-700`.
- **펼침**(테이블 표시 중): 위 화살표 + `데이터 테이블 숨기기`
- **숨김**(테이블 접힘): 아래 화살표 + `데이터 테이블 펼치기`
- 기본 상태는 **펼침**이다. 접으면 테이블 본문(래퍼·보충 설명 포함)만 숨긴다.

**데이터 테이블 복사**

- 아이콘 16×16px.
- 누르면 **같은 카드에 표시 중인 데이터 테이블**을 클립보드에 넣는다. 붙여넣기 시 표 형식(HTML)과 탭으로 구분된 텍스트(TSV)를 함께 제공하며, 환경에 따라 텍스트만 복사될 수 있다.
- 복사 대상에는 **보기·비율·평균 등 표에 보이는 값**이 포함되고, **`응답 수(명)` 열은 제외**한다. 펼치기/숨기기·보충 설명 문구는 포함하지 않는다.

## 3. 데이터 표시 규칙

### 비율과 반올림
- 비율은 내부 계산값을 유지하고, 표시 시점에만 소수점 첫째 자리까지 반올림한다.
- 화면에 쓸 때는 소수 첫째 자리로 반올림한다. 컴퓨터 계산 때문에 `33.299999…%`처럼 어긋나는 경우를 막기 위해, 반올림 직전에 아주 작은 값을 더한 뒤 반올림한다.
- 응답자 수 `N`은 반올림하지 않고 정수 그대로 표시한다.
- 같은 문항 안에서는 차트, 툴팁, 표가 동일한 비율 표시 규칙을 사용한다.
- 툴팁에서는 응답 비율을 `xx.x%`, 응답 수를 `N=xxx` 형식으로 표시한다.

### 보기 정렬
- 기본 보기 순서는 코드북의 보기 정의 순서를 따른다.
- 코드북에 없는 예외 응답이 있으면 기존 보기 뒤에 추가한다.
- 전체 기준으로 응답 0건인 보기는 기본 차트와 표에서 숨긴다.
- 그룹 비교에서도 전체 기준 0건 보기는 함께 숨긴다.

### 그룹핑 규칙
- 그룹 비교 시 그룹명은 `분석 기준 문항명: 그룹 값` 형식으로 표기한다. (예: `성별: 여`, `세대: Z세대`)
- 그룹별 표시·숨김, 범례 액션 버튼, 비활성 표시는 **`2-4. 범례 패널`**을 따른다.
- lane-group(그룹을 세로로 나열) 차트의 좌측 그룹명 레이블은 **`2-3-1. 가로 막대`** 보기명 열과 동일하다(`#### 2-3-12. 그룹별 비교` **(3)**).

### 기타 응답 보기
- 기타 응답용 텍스트 컬럼이 있으면 데이터 테이블의 해당 보기명 옆에 `응답 보기` 버튼을 표시할 수 있다.
- 버튼을 누르면 기타 자유응답 목록을 모달로 보여준다.
- 모달에는 중복 제거 없이 실제 수집된 텍스트 목록을 그대로 보여준다.

### 4-1. 객관식 단일

#### 집계 기준
- 각 보기별 선택 응답 수를 집계하고, 전체 응답 수 대비 비율을 계산한다.

#### 컨트롤바
- 그래프 유형 드롭다운 + 응답 비율 내림차순 정렬 옵션(체크박스 + 텍스트). 

#### 차트
- 기본 차트는 **가로 막대**. 그래프 유형 드롭다운으로 **세로 막대** / **100% 기준 누적 가로 막대** / **원형(파이)**로 전환할 수 있다(→ 위 차트 유형 옵션 요약 표).
- **가로 막대**·**세로 막대**는 각각 `#### 2-3-1`, `#### 2-3-4`를 따른다(단색 막대, `1-2` 단일 색상 그래프 팔레트).
- **100% 기준 누적 가로 막대**는 `#### 2-3-3`, **원형(파이)**는 `#### 2-3-8`을 따른다. 보기별 구간 색은 `1-2` 객관식 단일 — 100% 기준 누적 막대 / 원형 및 아래 범례와 같다.

#### 범례
- **가로 막대** / **세로 막대** / **100% 기준 누적 가로 막대** : 범례 없음 
- **원형(파이)** 차트: 범례 섹션에 색상 swatch + 보기명만 표시한다 (체크박스 없음)

#### 그룹 비교 시 차트 및 범례
- **가로 막대** / **100% 기준 누적 가로 막대**에서만 지원한다.

| 차트 | 차트 영역 | 우측 범례 패널 |
|------|-----------|----------------|
| **가로 막대** | 막대 좌측은 그대로 보기명 열. 행마다 전체 막대(`neutral-300`) 위에 그룹별 색 도트로 값을 겹쳐 표시 | 그룹 범례만 — 그룹명 + 체크박스 + `전체 선택`·`전체 해제`·`그룹 편집` 등(`2-4` 그룹 범례) |
| **100% 기준 누적 가로 막대** | 막대 좌측은 그룹명 열(`응답자 전체` 포함). 행마다 해당 그룹의 100% 누적 막대(`2-3-12` (3) lane-group) | 위 — 보기 범례(swatch + 보기명, 체크박스 없음) → 구분선 → 아래 — 그룹 범례(그룹명 + 체크박스 + 액션 버튼) |

- **가로 막대**에서 범례에 표시 중인 그룹이 두 개일 때 `두 그룹만 비교하기`로 이중 막대 모드로 전환할 수 있다. 켜진 뒤에는 같은 버튼이 `기본 그래프로 보기`로 바뀌어 겹침(도트) 모드로 되돌릴 수 있다. 레이아웃·간격·색 등은 `#### 2-3-12. 그룹별 비교` (2), 버튼 동작은 `2-4`를 따른다.

#### 툴팁
- 기본: `보기명`, `응답 비율`, `응답 수`
- 그룹 비교: `그룹명`, `보기명`, `응답 비율`, `응답 수`

#### 데이터 테이블
- 전체: `보기`, `응답 비율(%)`, `응답 수(명)`
- 그룹 비교: 전체 + 각 그룹의 `응답 비율(%)`, `응답 수(명)`

### 4-3. 객관식 순위

#### 집계 기준
- 순위 문항은 순위별 expanded 컬럼을 기준으로 집계한다.
- 순위 expanded 컬럼명은 `__1순위` 표준 형식을 우선 사용한다. legacy 호환을 위해 `_1순위` 형식도 인식한다.
- 응답자 수 `N`은 raw 값이 있거나 순위 expanded 값이 하나라도 있으면 포함한다.
- 순위별 비율 계산 모수는 해당 문항에 참여한 전체 응답자 수를 사용한다.

#### 가중 점수 계산
- 가중치는 최하위 순위부터 1점씩 시작해 순위가 올라갈수록 2씩 증가한다.
  - 예: 3순위까지인 경우 1순위=5점, 2순위=3점, 3순위=1점
  - 예: 4순위까지인 경우 1순위=7점, 2순위=5점, 3순위=3점, 4순위=1점
- 보기별 가중 평균 점수 = 각 순위 응답 수 × 해당 순위 가중치의 합 ÷ 전체 응답자 수
- 종합 순위는 가중 평균 점수 기준 내림차순. 동률이면 공동 순위로 표시한다.

#### 비순위·기타 응답
- 원문항 `raw`에 보기를 선택했지만 순위별 `expanded` 열에는 그 보기를 넣지 않은 응답(비순위 응답)이다. 실무적으로는 `기타`(직접입력) 보기의 기타 응답이 대부분이다. 전체 응답 수 `N`과 표·누적 막대 보기별 합계(응답 수·비율)에는 포함하되, 가중 평균 점수·종합 순위 계산에는 넣지 않는다.
- `기타` 보기는 순위 `expanded`에 값이 있어도 롤리팝·누적 막대·종합 순위에는 넣지 않는다. 표의 응답 수·순위별 비율에는 포함한다. 자유입력 텍스트는 `기타` 행 `응답 보기`로 본다(`## 3` 기타 응답 보기).

#### 컨트롤바
- 그래프 유형 드롭다운(가로 / 세로). 그룹 비교 시에는 세로를 선택할 수 없다.
- 그래프 형태 토글: `가중 평균` / `응답 비율`(각각 롤리팝·누적 막대에 대응, → 위 차트 유형 옵션 요약 표). 그룹 비교 시 `응답 비율`은 비활성이며 **가로 롤리팝**만 사용한다.
- 가중 평균 내림차순 정렬 옵션 (체크박스 + 텍스트).
- 가중 평균 산출 방법 안내 문구: 컨트롤바 하단 보충 영역에 표시한다. `각 보기의 가중 평균 점수는 순위별 응답 수에 가중치를 곱해 합산한 뒤 전체 응답자 수로 나눈 값입니다. 예) 3순위까지 선택하는 경우: (1순위×5 + 2순위×3 + 3순위×1) ÷ 전체 응답자 수 (만점: 5점)`

#### 차트
- 기본 차트는 **가로 롤리팝**(방향 `가로` + 형태 `가중 평균`). **세로 롤리팝**은 방향을 `세로`로 바꾼 뒤 형태 `가중 평균`일 때, **가로 누적 막대**는 방향 `가로` + 형태 `응답 비율`, **세로 누적 막대**는 방향 `세로` + 형태 `응답 비율`일 때 나온다.
- **가로 롤리팝**·**세로 롤리팝**: `#### 2-3-6`·`#### 2-3-7`을 따른다. `기타` 보기도 차트에는 표시되나, 종합 순위 산정에서만 제외한다(위 **비순위·기타 응답**).
- **가로 누적 막대**·**세로 누적 막대**: `#### 2-3-2`·`#### 2-3-5`, 구간 색 `1-2` 순위 계열. `기타` 보기도 차트에 표시된다.

#### 범례
- **가로 누적 막대**·**세로 누적 막대**만 **순위 범례** — `1순위`·`2순위`… swatch + 체크박스, `전체 선택`·`전체 해제`(`2-4` 순위 범례).
- **가로 롤리팝**·**세로 롤리팝**: 범례 없음.

#### 그룹 비교 시 차트 및 범례
- **가로 롤리팝**에서만 지원한다. 그룹이 있을 때 **세로** 방향·**응답 비율** 형태는 선택할 수 없다.

| 차트 | 차트 영역 | 우측 범례 패널 |
|------|-----------|----------------|
| **가로 롤리팝** | 보기명 열 유지. 행마다 전체 롤리팝 줄기(`neutral-300`) 위에 그룹별 **색 도트**로 가중 평균을 겹쳐 표시. 우측 종합 순위 열은 **응답자 전체** 기준 | 그룹 범례만 — 그룹명 + 체크박스 + `전체 선택`·`전체 해제`·`그룹 편집` 등(`2-4` 그룹 범례) | |

- **가로 롤리팝**에서 범례에 표시 중인 그룹이 두 개일 때 `두 그룹만 비교하기`로 이중 롤리팝 모드로 전환할 수 있다. 켜진 뒤에는 `기본 그래프로 보기`로 겹침(도트) 모드로 되돌릴 수 있다. 레이아웃·간격·색은 `#### 2-3-12. 그룹별 비교` (2), 버튼은 `2-4`를 따른다.

#### 툴팁
- **가로 롤리팝**·**세로 롤리팝**(그룹 없음): `보기명`, `가중 평균 점수`, `종합 순위`(응답자 전체 기준)
- **가로 누적 막대**·**세로 누적 막대** 구간: `보기명`, `해당 순위`, `응답 비율(%)`, `응답 수(명)`
- 그룹 비교 **가로 롤리팝**: `그룹명`, `보기명`, `가중 평균 점수`, `그룹 내 순위`(해당 그룹 안에서의 종합 순위)

#### 데이터 테이블
- 전체: `보기` → 순위별 `응답 비율(%)`·`응답 수(명)` → `가중 평균`·`종합 순위`. 하단 합계 행의 `가중 평균`·`종합 순위`는 `-`.
- 그룹 비교: `보기` → `응답자 전체` 블록(순위별 열 + 가중 평균·종합 순위) → 각 그룹 블록(동일 구조, 그룹별 `N`·가중 평균·종합 순위).
- `기타` 보기 행에 자유입력 텍스트가 있으면 `응답 보기` 버튼(`## 3` 기타 응답 보기).

#### 1순위만 보기
- 질문 제목 옆 `1순위만 보기` 버튼: 누르면 해당 문항의 1순위 응답만을 활용한 결과 카드로 교체된다. 차트 구현은 **객관식 단일**의 규칙을 따른다. 활성 시 버튼 문구는 `모든 순위 보기`로 바뀌며, 누르면 다시 원래 **객관식 순위** 문항 카드(롤리팝·누적 막대 등)로 돌아간다.

### 4-4. 객관식 척도

척도 문항은 `raw 척도`와 `derived 척도`를 구분해 처리한다.

#### raw 척도 집계 기준
- 각 점수별 응답 수를 집계하고, 전체 응답 수 대비 비율과 전체 평균값을 계산한다.

#### raw 척도 컨트롤바
- 그래프 모양 드롭다운: **가로 누적 막대** / **원 그래프** 전환(→ 위 차트 유형 옵션 요약 표). 그룹 비교 시 드롭다운은 없고 **가로 누적 막대**로 고정한다.
- 그래프 형태 토글 : `분포 보기` ↔ `평균 보기` 토글. **가로 누적 막대**일 때만 활성한다. **원 그래프** 선택 시 둘 다 비활성.
- 그룹 비교 시에만: `평균값이 높은 순서로 정렬` 체크박스. 체크 시 그룹 행을 그룹 평균 내림차순으로 정렬한다(`응답자 전체` 행 위치는 고정).
- 홀수 척도(3점 이상·홀수 길이)일 때만 `중간값 제외 보기` 체크박스. **원 그래프**에서는 비활성.
  - 체크 시 중립 점수 구간(예: 5점 척도 `3점`, 7점 척도 `4점`)을 차트에서만 숨긴다.
  - 숨긴 구간을 제외한 나머지 구간 너비만 100%로 다시 맞추며, 구간 안·표·툴팁의 응답 비율 수치는 원본 그대로다.
- 컨트롤바 하단 안내: `중간값 제외 보기는 차트에서만 시각적으로 제외하며, 응답 비율의 계산 모수와 원본 수치는 바뀌지 않습니다.`

#### raw 척도 차트
- 기본: **가로 누적 막대** + `분포 보기`.
- **가로 누적 막대** + `분포 보기`: `#### 2-3-3` 골격·구간 내부 수치 규칙을 따르고, 구간 색은 `1-2` 척도 계열. 구간 너비 6% 미만이면 구간 안 수치를 숨긴다. 막대 아래 좌·우에 `하위 척도 응답 합계`·`상위 척도 응답 합계`와 비율을 둔다(중립 점수는 합계에 넣지 않음). 합계 문구·수치는 `2-3-3` 하단 이중 행과 같이 `Label-1` / `Label-1-strong`, 줄 간격 0, `neutral-700`.
- **가로 누적 막대** + `평균 보기`: `#### 2-3-9. 척도 평균 그래프`. 코드북에 최저·최고점에 해당하는 보기 문구가 있으면 척도점 레이블 아래 좌·우 끝에 표시한다.
- **원 그래프**: `#### 2-3-8. 원형(파이)`, 구간 색 `1-2` 척도 계열.

#### raw 척도 범례
- **가로 누적 막대**·`분포 보기`, **원 그래프**: 점수 범례 — swatch + `N점` + (코드북 보기 문구), 체크박스 없음(`2-4` 점수 범례).
- `평균 보기`: 범례 없음.

#### raw 척도 그룹 비교 시 차트 및 범례
- **가로 누적 막대**(분포)와 **척도 평균 그래프**(평균) 모두 `#### 2-3-12. 그룹별 비교` (3) lane-group으로 `응답자 전체` → 구분선 → 그룹별 행을 세로로 나열한다. 행 간격 16px.
- `분포 보기`·`평균 보기` 모두 행마다 하단 레이블(끝점 보기명·`평균` 문구·하위/상위 합계 등)은 없고, 숫자만 남긴다. `분포 보기`는 막대 구간 안 응답 비율 수치는 그대로 표시하되, 하단의 하위/상위 척도 응답 합계 레이블은 숨긴다. `평균 보기`는 행별 척도점·끝점 레이블을 숨기고 차트 맨 아래 공통 축에 점수(1~N)만 표시한다.
- `평균값이 높은 순서로 정렬` 체크 시 그룹 행은 평균 내림차순으로 재배열된다. `응답자 전체` 행은 항상 최상단에 고정된다.

| 모드 | 차트 영역 | 우측 범례 패널 |
|------|-----------|----------------|
| 분포 보기 | 행마다 해당 그룹(또는 `응답자 전체`)의 누적 막대. 하위·상위 합계 없음 | 점수 범례 → 구분선 → 그룹 범례(`2-4`) |
| 평균 보기 | 행마다 음영 분포 막대 + 평균 원 마커 | 동일 |

#### raw 척도 툴팁
- **가로 누적 막대** + `분포 보기` 조각, **원 그래프** 조각: `N점`(보기 문구가 있으면 `N점 · 보기문구`), `응답 비율`, `응답 수`
- **가로 누적 막대** + `평균 보기` 평균마커 : `평균값`(소수 둘째 자리+`점`), `응답 수`
- 그룹 비교 · `분포 보기` 조각 : `그룹명`, `N점`(보기 문구가 있으면 `N점 · 보기문구`), `응답 비율`, `응답 수`
- 그룹 비교 · `평균 보기` 평균마커 : `그룹명`, `평균값`(소수 둘째 자리+`점`), `응답 수`

#### raw 척도 데이터 테이블
- 전체: `점수`, `응답 비율(%)`, `응답 수(명)` | `평균` — `응답 수(명)` 열과 `평균` 열 사이에 1px `neutral-300` 구분선(`group-col`, `2-5-3`). `평균` 값은 합계 행에만 표시.
- 그룹 비교: `점수` → `응답자 전체`·각 그룹 블록(`응답 비율(%)`, `응답 수(명)` | `평균`). 블록마다 `응답 수(명)`과 `평균` 사이 동일 구분선. `평균`은 각 블록 합계 행에만 표시. `평균값이 높은 순서로 정렬` 체크 시 그룹 블록 순서가 평균 내림차순으로 바뀐다. `응답자 전체` 블록은 항상 첫 번째.

#### derived 척도 집계 기준
- derived 열의 연속 점수를 모아 최소·최대·사분위·평균·응답 수를 계산한다.

#### derived 척도 컨트롤바
- 기본적으로 컨트롤 요소는 없고, 박스플롯에 대한 안내 문구만 표시한다. `박스플롯 차트 보는 법: 수염은 응답값의 전체 범위, 박스는 전체 응답 중 가운데 50%가 모인 구간, 박스 중앙 마커(Q2)는 중앙값입니다.`
- 그룹 비교 시에는 `평균값이 높은 순서로 정렬` 체크박스가 추가된다. 체크 시 그룹 행을 그룹 평균 내림차순으로 정렬한다(`응답자 전체` 행 위치는 고정).

#### derived 척도 차트
- 기본 차트는 **박스플롯**(`2-3-10`). 가로축은 `1`~`N`점 정수 눈금(`N` = 척도 길이).
- whisker·박스(Q1~Q3)·최솟값/최댓값 tick, 평균 원 마커는 `2-3-10`을 따른다. 평균 마커 안 숫자는 소수 둘째 자리까지 표시한다.
- `Q1`·`Q2`·`Q3` 이름·값은 박스 아래에 표시한다. Q 블록 클릭 시 강조/연하게 처리(`2-3-10`).

#### derived 척도 범례
- 없음

#### derived 척도 그룹 비교 시 차트 및 범례
- 박스플롯을 lane-group으로 세로 나열한다. `응답자 전체` → 구분선 → 그룹별 행(좌측 그룹명 레이블). 맨 아래 공통 `1`~`N`점 축.
- 우측 범례: **그룹 범례**만(그룹별 체크박스, `전체 선택`·`전체 해제`·`그룹 편집`). 점수 범례 없음.
- `평균값이 높은 순서로 정렬` 체크 시 그룹 행은 평균 내림차순으로 재배열된다. `응답자 전체` 행은 항상 최상단에 고정된다.

#### derived 척도 툴팁
- 최솟값 tick: `최솟값`, 해당 값(소수 둘째 자리)
- 최댓값 tick: `최댓값`, 해당 값(소수 둘째 자리)
- Q1 tick·Q1 블록: `Q1(하위 25%)`, 해당 값(소수 둘째 자리)
- Q2 tick·Q2 블록: `Q2(중앙값)`, 해당 값(소수 둘째 자리)
- Q3 tick·Q3 블록: `Q3(상위 25%)`, 해당 값(소수 둘째 자리)
- 평균 마커: `평균값`(소수 둘째 자리), `응답 수`
- 그룹 비교 시 각 툴팁 맨 위 행에 `그룹명` 추가

#### derived 척도 데이터 테이블
- 전체: `문항명`, `최소값`, `Q1(하위 25%)`, `Q2(중앙값)`, `Q3(상위 25%)`, `최대값` | `평균`, `응답 수(명)` (헤더 제외 단일 행 표) — `최대값`과 `평균` 사이, `평균`과 `응답 수(명)` 사이에 각각 1px `neutral-300` 구분선(`group-col`).
- 그룹 비교: `그룹명`, `최소값`, `Q1(하위 25%)`, `Q2(중앙값)`, `Q3(상위 25%)`, `최대값` | `평균`, `응답 수(명)` — 표시 중인 그룹별 행, 맨 아래 `응답자 전체` 합계 행(`total-row`). 동일 구분선 적용. `평균값이 높은 순서로 정렬` 체크 시 그룹 행 순서가 평균 내림차순으로 바뀐다. `응답자 전체` 행은 항상 맨 아래에 고정된다.

#### 척도형 여러 문항 비교

- 문항 사이드 패널 `보고 싶은 문항`에 올린 **문항 칩이 모두 객관식 척도이고, 척도 길이가 전부 같으며, 2개 이상**일 때만 활성화된다(`raw`·`derived` 모두 포함). 길이가 다른 문항이 하나라도 있거나, 척도가 아닌 문항이 섞이면 버튼이 비활성이다.
- 활성화 후 버튼으로 개별 문항 보기 ↔ 여러 문항 한 번에 비교 하기를 전환할 수 있다. 여러 문항 비교 시 `보고 싶은 문항`에 올린 문항이 모두 나온다. 문항마다 좌측 `문항명` + 차트 한 줄(단일 문항 그룹 비교처럼 `응답자 전체`·그룹을 행으로 쌓지 않음).

#### 척도형 여러 문항 비교 - 컨트롤바

- `분포 보기` ↔ `평균 보기` 토글. 단, **그룹별 비교 기준**이 있으면 `분포 보기`는 쓸 수 없고 `평균 보기`로 고정된다.
- `평균값이 높은 순서로 정렬`(체크박스 + 텍스트)
- 비교 문항에 `raw 척도`가 있으면 `중간값 제외 보기`(차트만, 수치는 그대로).

#### 척도형 여러 문항 비교 - 차트

- **기본** — `분포 보기`: `raw`는 100% 누적 막대, `derived`는 박스플롯. `평균 보기`: `raw`는 척도 평균 그래프, `derived`는 박스플롯(평균 마커 있음, 박스·수염선·tick 등 나머지 요소는 opacity 0.3으로 흐리게 처리). `평균 보기`만 맨 아래 공통 `1`~`N`점 축. 우측 점수 범례.
- **그룹별 비교 시** — `평균 보기`만 지원한다. 문항마다 배경 트랙(`raw` 음영 분포 / `derived` 박스플롯, **평균 원 마커 없음**) 위에 평균 도트를 겹친다. `neutral-700` = `응답자 전체`, 그룹 색 = 그룹 평균(도트 옆 수치). 맨 아래 공통 `1`~`N`점 축. 우측 그룹 범례.
- **두 그룹만 비교하기** — 그룹 범례에서 체크해 둔 그룹이 **정확히 2개**일 때만 범례 아래 버튼이 나타난다. 누르면 위의 배경 트랙·겹침 도트 대신, 문항마다 두 그룹의 평균을 **가로 롤리팝 두 줄**(그룹 색 줄기·도트·평균 수치)로 나란히 본다. 다시 누르면(`기본 그래프로 보기`) 겹침 도트 보기로 돌아간다.

#### 척도형 여러 문항 비교 - 데이터 테이블

- 기본: `문항명` + 점수별 `응답 비율(%)`·`응답 수(명)` + `평균` | `전체 응답 수(명)`. `derived`는 점수 칸 `-`. `평균`과 `전체 응답 수(명)` 사이 1px `neutral-300` 구분선(`group-col`).
- 그룹별 비교 시: `문항명` + `응답자 전체`·각 그룹의 `평균`·`응답 수(명)`. 구분선 없음.

### 4-5. 주관식 숫자

#### 집계 기준
- 숫자 응답값을 기준으로 최소·최대·사분위(`Q1`, `Q2`, `Q3`)·평균·응답 수를 계산한다.
- 히스토그램 보기에서는 구간별 응답 수와 비율을 계산한다.

#### 컨트롤바
- `요약 보기` ↔ `분포 보기` 토글. 박스플롯 ↔ 히스토그램 전환 가능. 그룹 비교 시 비활성(박스플롯 고정).
- 구간 시작값·구간 간격 입력 필드: `분포 보기`(히스토그램) 선택 시에만 활성화된다.
- 하단 안내 문구: `요약 보기` 시 `박스플롯 차트 보는 법: 수염은 응답값의 전체 범위, 박스는 전체 응답 중 가운데 50%가 모인 구간, 박스 중앙 마커(Q2)는 중앙값입니다.` / `분포 보기` 시 `구간 시작값과 간격을 조정해서 차트를 조정해보세요. 참고로 각 구간은 시작값 '이상' 다음 경계값 '미만'을 의미하며, 마지막 구간은 최댓값을 포함합니다.`

#### 차트
- 기본 차트는 **박스플롯**. `요약 보기` ↔ `분포 보기` 토글로 **히스토그램**으로 전환할 수 있다. 그룹 비교 시 박스플롯 고정.
- **박스플롯**은 `#### 2-3-10. 박스플롯`을 따른다.
- **히스토그램**은 `#### 2-3-11. 히스토그램`을 따른다.

#### 범례
- 없음

#### 그룹 비교 시 차트 및 범례
- 박스플롯을 lane-group으로 세로 나열한다. `응답자 전체` → 구분선 → 그룹별 행(좌측 그룹명 레이블). 맨 아래 공통 축.
- 우측 범례: **그룹 범례**만(그룹별 체크박스, `전체 선택`·`전체 해제`·`그룹 편집`).

#### 툴팁
- 박스플롯 최솟값/최댓값 tick: `최솟값`/`최댓값`, 해당 값
- 박스플롯 Q1/Q2/Q3 tick·블록: `Q1(하위 25%)`/`Q2(중앙값)`/`Q3(상위 25%)`, 해당 값
- 평균 마커: `평균값`, `응답 수`
- 히스토그램 막대: `구간 범위`, `응답 비율(%)`, `응답 수`
- 그룹 비교 시 각 툴팁 맨 위 행에 `그룹명` 추가

#### 데이터 테이블
- 전체: `문항명`, `최소값`, `Q1(하위 25%)`, `Q2(중앙값)`, `Q3(상위 25%)`, `최대값` | `평균`, `응답 수(명)` (헤더 제외 단일 행 표) — `최대값`과 `평균` 사이, `평균`과 `응답 수(명)` 사이에 각각 1px `neutral-300` 구분선(`group-col`).
- 그룹 비교: `그룹명`, `최소값`, `Q1(하위 25%)`, `Q2(중앙값)`, `Q3(상위 25%)`, `최대값` | `평균`, `응답 수(명)` — 그룹별 행 → 맨 아래 `응답자 전체` 합계 행(`total-row`). 동일 구분선 적용.

### 4-6. 주관식 비율 배분

#### 집계 기준
- 항목별 평균 배분값을 계산한다.
- 항목별 합계가 100인 응답만 유효 응답으로 집계한다.

#### 컨트롤바
- 그래프 모양 선택 드롭다운: `가로 누적 막대`(기본) / `원형(파이)`. 그룹 비교 시 숨김.
- 안내 문구 항상 표시: `각 응답자가 두 항목의 합이 100이 되도록 값을 나누어 기입하는 문항입니다. 아래 차트는 각 항목에 기입한 값의 평균을 보여줍니다.` 유효하지 않은 응답이 있으면 제외 건수를 문구 뒤에 추가한다.

#### 차트
- 기본 차트는 **가로 100% 누적 막대**. 그래프 모양 드롭다운으로 **원형(파이)**로 전환할 수 있다. 그룹 비교 시 가로 100% 누적 막대 고정.
- **가로 100% 누적 막대**: 각 구간 너비는 항목별 평균 배분값을 나타낸다. 막대·구간의 치수·배치 골격은 `#### 2-3-3. 가로 막대 100% 기준 누적형`을 따른다. 각 막대 하단에 항목별 레이블과 평균 배분값을 표시한다.
- **원형(파이)**: `#### 2-3-8. 원형(파이)` 참조.

#### 범례
- **가로 100% 누적 막대**: 범례 없음(차트 하단 레이블로 표시).
- **원형(파이)**: 항목 범례 — 색상 swatch + 항목명, 체크박스 없음.
- 항목 색상은 `color-1`부터 항목 순서대로 적용한다.

#### 그룹 비교 시 차트 및 범례
- 가로 100% 누적 막대를 lane-group으로 세로 나열한다. `응답자 전체` → 구분선 → 그룹별 행(좌측 그룹명 레이블). 그룹 행에는 하단 레이블 없음.
- 우측 범례: **항목 범례**(색상 swatch + 항목명, 체크박스 없음) → 구분선 → **그룹 범례**(그룹별 체크박스, `전체 선택`·`전체 해제`·`그룹 편집`).

#### 툴팁
- `배분 항목`, `평균 비중(%)`, `응답 수`
- 그룹 비교 시 맨 위 행에 `그룹명` 추가

#### 데이터 테이블
- 전체: `배분 항목`, `평균 배분값(%)`, `응답 수(명)`
- 그룹 비교: 전체 + 각 그룹의 `평균 배분값(%)`, `응답 수(명)`

### 4-8. 주관식 문자

#### 집계 기준
- 비어 있지 않은 텍스트 응답을 유효 응답으로 본다.
- 응답 텍스트는 중복 제거 없이 실제 수집된 값을 그대로 표시한다.

#### 컨트롤바
- 키워드 검색 input을 제공한다.
- 사용자가 키워드를 입력하면 해당 키워드를 포함하는 응답만 즉시 필터링한다.
- 검색어가 비어 있으면 전체 유효 응답을 표시한다.

#### 응답 목록
- 차트 대신 스크롤 가능한 응답 목록 박스를 표시한다.
- 박스 상단에는 `응답 {검색 결과 수}건 / 전체 {전체 유효 응답 수}명` 형식의 요약을 표시한다.
- 박스 본문에는 각 응답을 한 줄 또는 문단 단위로 세로 나열한다.
- 응답이 많으면 박스 내부에서만 세로 스크롤한다.

#### 툴팁
- 없음

#### 범례
- 없음

#### 그룹 비교
- 지원하지 않는다.

#### 데이터 테이블
- 별도 데이터 테이블은 표시하지 않는다.

## 5. 상호작용 규칙

### 그룹 on/off
- 그룹 비교 차트에서 표시할 그룹을 사용자가 직접 선택할 수 있다.
- 선택 해제된 그룹은 범례에서는 흐리게 표시되고, 차트와 표에서는 숨겨진다.
- 체크 상태 변경은 현재 표시 중인 차트와 데이터 테이블에 즉시 반영한다.

### 그래프 유형 전환
- 드롭다운을 사용하는 차트 유형에서는, 문항 형식별로 허용된 그래프 유형 옵션을 선택하면 즉시 재렌더링한다.
- 정렬 변경은 현재 선택된 그래프 유형에 동일하게 적용한다.

## 6. 기타 응답 처리 규칙

### 기타 텍스트 컬럼 탐색
- 기타 자유응답 텍스트는 전용 expanded 컬럼에서 읽는다.
- 현재 구현은 아래와 같은 이름 패턴을 순차적으로 허용한다.
  - `__기타_텍스트`
  - `__기타 텍스트`
  - `_기타_텍스트`
  - `_텍스트`
- 위 패턴에 정확히 맞지 않아도, 문항명과 기타 텍스트 의미가 함께 드러나는 경우 느슨한 fallback 탐색을 수행한다.

### 기타 응답 표시
- 데이터 테이블에서 기타 보기 항목 옆에 `응답 보기` 버튼을 표시한다.
- 모달에서는 중복 제거 없이 실제 응답 텍스트 목록을 보여준다.
- 기타 응답은 차트 표시보다 상세 확인 보조 기능에 가깝게 다룬다.

## 7. 예외 및 비지원 처리

### 데이터 부족
- 문항은 지원 유형이더라도 실제 집계에 필요한 컬럼이 없으면 결과를 그리지 못할 수 있다.

### 순위 문항 컬럼 불완전
- 순위형은 순위별 expanded 컬럼을 찾지 못하면 정상 집계를 수행하지 않는다.
- raw 값만 있다고 해서 순위 차트를 강제로 생성하지 않는다.

### 중복 문항 컬럼 불완전
- 중복형은 보기별 expanded 이진 컬럼을 찾지 못하면 정상 집계를 수행하지 않는다.
