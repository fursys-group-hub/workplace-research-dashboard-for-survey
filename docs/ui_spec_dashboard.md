최종 갱신: 2026-06-02
대상 파일: dashboard.html

---

## 1. 화면 개요 / 목적

### 1.1 화면 정보
- 화면명: 설문조사 분석 대시보드
- 파일: dashboard.html
- 브라우저 탭 제목: 설문조사 분석 대시보드 | purple6studio

### 1.2 목적
표준화된 설문 데이터를 별도의 도구 없이 바로 시각화해, 
문항별 분석 결과를 빠르게 확인하고 필요 시 보고서에 활용할 수 있도록 한다.

> 차트 영역의 시각화 종류·동작·옵션은 `data_visualization.md`를 참조한다.

### 1.3 사용자 여정
1. 홈에서 파일을 업로드하거나 저장된 대시보드를 열면 이 화면으로 진입한다.
2. 좌측 문항 리스트를 탐색하거나 검색해 원하는 문항을 찾는다.
3. 문항을 드래그해 **보고 싶은 문항** 존에 올린다 → 차트가 자동 생성된다.
4. 필요하면 필터를 추가하거나, 그룹별 비교 문항을 지정한다.
5. 차트 이미지를 저장해 보고서에 활용한다.

## 2. 레이아웃 구조

### 2.1 전체 레이아웃
화면은 다음 5개 영역으로 구성된다.

| 영역 | 설명 |
| --- | --- |
| 헤더 | 로고, 설문 제목, 액션 버튼으로 구성. 화면 상단에 고정 |
| 좌측 패널 | 문항 리스트. 화면 좌측에 고정 |
| 필터 바 | 응답자 필터 설정 버튼. 메인 영역 상단에 고정 |
| 메인 영역 | 문항별 결과 카드(차트, 범례, 데이트 테이블 등). 이 영역만 스크롤됨 |
| 푸터 | 현재 비어 있음. 화면 하단에 고정. 추후 일괄 추출 기능이 생기면 이 영역에 버튼 생성 예정 (TBD) |

### 2.2 페이지 구조

```
헤더
본문
  좌측 패널
  메인 영역
    필터 바
    분석 구조   보고 싶은 문항 카드 / 그룹별 비교 카드
    결과 영역   문항별 차트 + 데이터 테이블
푸터
```

## 3. UI 컴포넌트 상세 스펙

### 3.1 헤더

#### 로고
- purple6studio 로고
- 클릭 시 home.html로 이동

#### 설문 제목
- sessionStorage의 `survey.title` 값을 표시한다.
- 제목 우측 편집 아이콘 클릭 시 인라인 입력 필드로 전환된다.
  - Enter 또는 포커스 아웃 시 저장 / Escape 시 취소
  - 저장 시 Supabase DB의 해당 설문 title 업데이트
  - 최대 50자

#### 액션 버튼 (우측, 3개)
| 버튼 | 동작 |
| --- | --- |
| 데이터 교체하기 | 데이터 교체 모달 열기 |
| 저장된 대시보드 리스트 | 리스트 모달 열기. 저장된 개수 뱃지 표시 |
| 새 대시보드 만들기 | home.html로 이동 (검정 배경 강조 버튼) |

---

### 3.2 좌측 문항 패널

문항을 탐색하고 드래그하는 영역이다.

#### 문항 검색
- 검색어 입력 시 `category_1`, `category_2`, `question_label`, `question_full`, `question_no`를 대상으로 즉시 필터링한다.
- 매칭된 문항이 포함된 아코디언이 자동 확장된다.
- 결과 없을 때: `검색 결과가 없습니다.` 표시
- 검색어를 지우면 아코디언이 모두 닫힌다.

#### 문항 트리
- 코드북의 `category_1` → 1차 아코디언, `category_2` → 2차 아코디언으로 분류된다.
- 데이터 미로드 시: `새 대시보드를 만들거나 저장된 대시보드를 열어주세요` 안내 표시
- 로딩 중: `코드북을 불러오는 중입니다...` 표시

#### 문항 카드
- 카드를 드래그해 보고 싶은 문항 존 또는 그룹별 비교 존에 올린다.
- 이미 드롭존에 올라간 문항은 선택 상태(테두리 강조)로 표시된다.
- 패널 확장(우측 핸들 클릭) 시 문항명과 문항 전체 텍스트를 나란히 볼 수 있다.

#### 선택 상태 바
- 드롭존에 문항이 1개 이상 배치되면 패널 하단에 `N개 선택됨` + `전체 해제` 버튼이 나타난다.

---

### 3.3 필터 바

분석 대상 응답자를 조건으로 좁히는 영역이다. 메인 영역 최상단에 고정된다.

- **+ 필터 추가** 버튼 클릭 → 드롭다운에서 문항 선택 → 보기 체크박스로 조건 지정
- 추가된 필터는 칩 형태로 나열되며, 드래그로 순서를 바꿀 수 있다.
- 칩 우측 × 클릭으로 필터 제거
- 칩 클릭 시 해당 필터의 조건 드롭다운 재오픈
- 우측 **N = 숫자**는 현재 필터 조건을 만족하는 응답자 수를 실시간으로 표시한다.

---

### 3.4 분석 구조 정의 영역

#### 보고 싶은 문항 카드
- 좌측 문항 카드를 드래그해 올리면 해당 문항의 차트가 결과 영역에 생성된다.
- 최대 20개까지 올릴 수 있다.
- 빈 상태: `좌측 문항을 여기로 드래그해 주세요.`
- **여러 문항 한 번에 비교하기** 버튼: 올라간 문항이 모두 같은 척도 길이의 척도 문항이고 2개 이상일 때 활성화된다. 클릭 시 척도 비교 모달 오픈.
- **모두 삭제** 버튼: 올라간 문항 칩 전체 제거

#### 그룹별 비교 카드
- 최대 1개. 새 문항을 올리면 기존 문항을 교체한다.
- 빈 상태: `좌측 문항을 여기로 드래그해 주세요.`
- **연도별 비교하기** 버튼: 데이터에 여러 연도가 있을 때 나타난다. 클릭 시 `survey_year`를 기준 문항으로 자동 설정.
- **삭제** 버튼: 올라간 문항 칩 제거

---

### 3.5 결과 영역

- 보고 싶은 문항에 칩이 없으면 `보고 싶은 문항을 드래그하면 차트가 생성됩니다` 안내를 표시한다.
- 문항이 올라오면 문항 유형에 맞는 차트와 데이터 테이블을 자동 렌더링한다.
- 차트 종류·동작·옵션은 `data_visualization.md` 참조.
- 각 결과 섹션 우측 상단에 **이미지로 저장** / **캡처** 버튼이 있다 (`image-export-spec.md` 참조).

### 3.6 모달 5종

| 모달 | 트리거 | 내용 |
| --- | --- | --- |
| 저장된 대시보드 리스트 | 헤더 리스트 버튼 | 저장된 대시보드 목록. 클릭 시 해당 대시보드 열기 / 이름 변경 / 삭제 가능 |
| 데이터 교체하기 | 헤더 데이터 교체 버튼 | 코드북·숫자형·텍스트형 파일을 각각 교체. **대시보드 업데이트** 버튼 클릭 시 3종 검증 후 반영 |
| 기타 응답 | 결과 테이블의 기타 항목 `응답 보기` 버튼 | 기타 자유응답 목록 표시 (작은 팝오버형, 백드롭 없음) |
| 척도 비교 | 보고 싶은 문항 카드의 `여러 문항 한 번에 비교하기` 버튼 | 같은 척도 길이 후보 문항 다중 선택 후 적용 |
| 범례 그룹 편집 | 결과 영역 범례의 `그룹 편집` 버튼 | 그룹별 비교 시 보기 항목들을 사용자 정의 묶음으로 재구성 |

#### 모달 공통 동작
- 닫기: × 버튼 / 배경 클릭 / Esc 키
- 저장된 대시보드 리스트: 이름 변경 시 인라인 입력 전환 → Enter/포커스 아웃 저장 / Esc 취소

## 4. 데이터 연동

- 인증: 페이지 로드 시 Supabase 세션 확인. 세션이 없거나 24시간 경과 시 login.html로 이동.
- 진입 시 sessionStorage의 `survey.currentId`로 Supabase에서 해당 설문 메타·CSV 파일을 불러온다.
- CSV 파일(코드북·숫자형·텍스트형)은 Supabase Storage(`shared/{id}/`)에서 다운로드해 인메모리로 처리한다.
- 설문 제목 변경·파일 교체는 Supabase DB/Storage에 즉시 반영된다.
