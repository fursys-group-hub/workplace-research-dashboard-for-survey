# 차트 이미지 추출 기능 명세

## 1. 목적

대시보드에서 생성된 차트를 고객사 제출용 **A4 보고서 페이지 이미지**로 추출한다.
내부 분석용 UI 컨트롤(보기 방식 전환, 정렬 옵션 등)은 이미지에서 제외하고,
문항 정보 + 차트 본문 + 푸터로 구성된 클린한 A4 한 페이지 이미지를 출력한다.

---

## 2. 출력 사양

| 항목 | 값 |
|---|---|
| 페이지 크기 | A4 세로 (210 × 297 mm) |
| 해상도 | 300 DPI |
| 픽셀 크기 | 2480 × 3508 px |
| 포맷 | PNG |
| 배경 | 흰색 (`#ffffff`) |
| 파일명 | `{문항 제목}.png` |

> 페이지 분할 없음. 모든 추출 이미지는 **A4 한 장**으로 완성된다.

---

## 3. 추출 트리거

- 각 `result-section`마다 **"이미지로 저장" 버튼**을 개별 배치한다.
- 버튼 위치: `.result-header` 영역 우측
- 버튼은 이미지에 포함되지 않는다 (캡처 전 일시 숨김 처리).

---

## 4. 이미지 구성

A4 한 페이지는 **헤더 / 콘텐츠 / 푸터** 3개 영역으로 구성된다.

### 4-1. 페이지 레이아웃 (mm 기준)

| 영역 | 크기 | 비고 |
|---|---|---|
| 페이지 전체 | 210 × 297 mm | A4 세로 |
| 좌우 여백 | 10 mm × 2 | 콘텐츠 가용 폭 **190 mm** |
| 상단 여백 | 10 mm | — |
| 콘텐츠 영역 | 180 × 272 mm | 헤더(문항 이름 + Q. 원문) + 차트 + 부속 요소 (데이터 테이블 등). 캡처된 `.result-section` 전체가 이 영역에 비례 축소되어 들어감 |
| 푸터 영역 | 180 × 15 mm | 가로 구분선 + 로고 + 카피라이트. 페이지 하단 가장자리까지 사용 (별도 하단 여백 없음) |

### 4-2. 헤더

기존 대시보드 화면의 스타일을 그대로 사용한다.

| 요소 | CSS 클래스 | 스타일 |
|---|---|---|
| 문항 이름 | `.result-question-label` | `Heading-4` (18px Bold), `neutral-900` |
| Q. 질문 원문 | `.result-question-full` | `Body-2` (16px Medium), `neutral-700` |

### 4-3. 콘텐츠

| 요소 | CSS 클래스 / 설명 |
|---|---|
| 차트 본체 | `.single-hbar-chart`, `.vbar-chart`, `.pie-chart-wrap`, `.stacked-chart-wrap`, `.lollipop-chart` 등 차트 영역 전체 |
| 가중 평균 설명 주석 | `.result-note` (산출식 설명 텍스트) |
| 범례 | `.result-side-panel`, `.result-legend` 등 범례 영역 (그룹 비교 적용 시 그룹 범례 포함) |
| 데이터 테이블 | `[data-data-table-section]` — **펼쳐진 상태인 경우에만** 포함 |

> 콘텐츠 영역의 차트와 부속 요소는 **현재 웹 화면의 폰트·색상·간격을 그대로 유지한 채 비례 축소**해서 콘텐츠 가용 폭(180 mm)에 맞춘다. 별도의 레이아웃 재구성·폰트 재계산은 하지 않는다.

### 4-4. 푸터

A4 추출 이미지 하단에 고정으로 들어가는 푸터 영역. 기존 대시보드 화면의 `.dash-footer`(전체 PPT 내보내기 버튼 영역)나 `.main-copyright`(중앙 정렬 카피라이트)와는 별개의, **추출 이미지 전용 푸터**다.

#### 영역 정의

| 항목 | 값 | 비고 |
|---|---|---|
| 위치 | 페이지 하단 10 mm | 페이지 하단 가장자리까지 사용 (별도 하단 여백 없음) |
| 가로 폭 | 190 mm | 좌우 여백 10 mm 적용 |
| 좌우 패딩 | 10 mm | 페이지 좌우 여백과 동일 |

#### 구성 요소

푸터 영역 안에는 다음 3가지 요소가 위치한다.

| 요소 | 위치 | 설명 |
|---|---|---|
| ① 가로 구분선 | 푸터 영역 상단 (= 콘텐츠 영역 종료점) | 콘텐츠 영역과 푸터를 시각적으로 구분 |
| ② 로고 | 푸터 영역 좌측, 세로 중앙 정렬 | `assets/purple6studio_한줄_black.png` |
| ③ 카피라이트 텍스트 | 푸터 영역 우측, 세로 중앙 정렬 | `© 2026 퍼플식스 스튜디오 (주)퍼시스. All rights reserved` |

#### ① 가로 구분선 스타일

| 항목 | 값 | 비고 |
|---|---|---|
| 위치 | 푸터 영역 상단 (페이지 하단에서 15 mm 위) | — |
| 가로 폭 | 190 mm (좌우 패딩 10 mm씩 적용) | — |
| 두께 | 2 px (≈ 0.17 mm @ 300 DPI) | — |
| 색상 | `--neutral-200` (`#e3e3e3`) | — |

#### ② 로고 스타일

| 항목 | 값 | 비고 |
|---|---|---|
| 자산 경로 | `assets/purple6studio_한줄_black.png` | — |
| 정렬 | 좌측 + 세로 중앙 | — |
| 높이 | 60 px (≈ 5 mm @ 300 DPI) | **사용자 지정 가능** — 푸터 영역(15 mm) 안에 들어가는 범위에서 조정 |
| 너비 | 자동 (원본 비율 유지) | 높이 변경 시 비율로 자동 계산 |

#### ③ 카피라이트 텍스트 스타일

| 항목 | 값 | 비고 |
|---|---|---|
| 텍스트 | `© 2026 퍼플식스 스튜디오 (주)퍼시스. All rights reserved` | — |
| 폰트 사이즈 (CSS 기준) | `--caption-1-size` (12 px) | 기존 `.main-copyright`와 동일 |
| 폰트 사이즈 (캔버스 픽셀) | 38 px (= 12 × 300 / 96) | **사용자 지정 가능** — 96 DPI 기준 12 px을 300 DPI 캔버스에서 동일 시각 크기로 환산한 값 |
| 폰트 굵기 | Regular (400) | `Caption-1` 토큰 |
| 라인 높이 | `--body-line-height` (1.6) | 기존 `.main-copyright`와 동일 |
| 색상 | `--Black` (`#000000`) | 디자인 시스템 토큰 중 가장 진한 검정 |
| 정렬 | 우측 + 세로 중앙 | — |

## 5. 이미지에서 제외할 요소

내부 분석 도구에 해당하는 UI 컨트롤은 캡처 직전 일시적으로 숨기고, 캡처 완료 후 복원한다.

| 요소 | CSS 클래스 / 속성 | 설명 |
|---|---|---|
| 그래프 유형 드롭다운 + 정렬 버튼 | `.chart-view-controls` | 가로/세로/원형/누적 전환, 기본순서/응답률 높은 순 |
| 순위 문항 컨트롤 전체 | `.choice-controls` | 가중 평균/응답 비율 탭, 막대 방향 탭, 가중 평균 높은 순서 정렬 체크박스 |
| 데이터 테이블 토글 버튼 | `[data-data-table-toggle]` | "데이터 테이블 펼치기/숨기기" 버튼 — 테이블 펼침 여부와 무관하게 항상 제외 |
| 이미지 저장 버튼 자체 | `.result-export-btn` | 캡처 버튼이 이미지에 찍히지 않도록 |

## 6. 레이아웃 처리

### 6-1. 비례 축소 룰

- 캡처 대상(`.result-section`)을 **현재 웹 화면에 보이는 그대로** 캡처한 뒤, 콘텐츠 영역 폭(180 mm = 2126 px @ 300 DPI)에 맞춰 **균등 비례 축소**한다.
- 폰트, 차트, 데이터 테이블 등은 모두 동일 비율로 축소되며, 별도의 폰트 크기 재계산이나 차트 재렌더는 하지 않는다.
- 축소율 = `2126 / 캡처원본너비`

### 6-2. A4 한 페이지 보장

A4 한 페이지(콘텐츠 영역 247 mm)에 모든 콘텐츠가 들어가야 한다. 비례 축소 후에도 콘텐츠 높이가 콘텐츠 영역을 초과하면 다음 룰을 따른다.

1. **데이터 테이블이 펼쳐진 상태인 경우**
   - 사용자에게 모달 노출: *"추출할 수 있는 페이지 영역을 초과합니다. 데이터 테이블을 숨기고 추출할까요?"*
   - 확인 → 데이터 테이블을 일시 숨김 처리한 뒤 다시 캡처·축소 → 결과 이미지 출력
   - 취소 → 추출 중단

2. **데이터 테이블이 이미 숨겨진 상태인데도 영역을 초과하는 경우**
   - 미결 (§ 8 참고)

### 6-3. 페이지 분할

페이지 분할은 **하지 않는다.** 모든 추출 결과는 단일 PNG 파일이다.

## 7. 구현 방식

- **라이브러리**: `dom-to-image-more` (SVG 처리 정확도 우수)
- **폰트**: 로컬 `assets/fonts/Pretendard-*.otf` 참조 (CDN CORS 문제 회피)
- **캡처 흐름**:
  1. 제외 대상 요소(`[data-data-table-toggle]`, `.chart-view-controls`, `.choice-controls`, `.result-export-btn`)를 `visibility: hidden` 처리
  2. 데이터 테이블이 **접힌 상태**라면 `[data-data-table-section]`도 추가로 숨김
  3. `document.fonts.ready` 대기
  4. `domtoimage.toCanvas(section, { bgcolor: '#ffffff' })`로 캡처 대상의 원본 픽셀 캡처
  5. § 6-1 룰에 따라 콘텐츠 영역 폭(2126 px)에 맞춰 비례 축소
  6. 축소 후 높이가 콘텐츠 영역 높이(2917 px)를 초과하면 § 6-2 분기 적용
  7. A4 캔버스(2480 × 3508 px) 위에 합성:
     - 좌우 여백 15 mm, 상단 여백 10 mm
     - 헤더(문항 이름 + Q. 질문 원문) → 콘텐츠 영역(축소된 차트 + 부속 요소) → 푸터(로고 + 카피라이트)
  8. PNG로 저장 (`{문항 제목}.png`)
  9. 제외했던 요소 복원

## 8. 미결 사항 (구현 전 결정 필요)

- [ ] **헤더의 "문항 이름" 대체 케이스 처리**
  - 일반 단일 문항이 아닌 경우(예: `targetScaleCompareMode` 상태에서 `.result-question-label`이 실제 문항명 대신 "여러 문항 한 번에 비교하기"로 대체되는 케이스), 추출 이미지의 헤더에 어떤 텍스트를 표시할지 정의 필요.
  - 가능한 옵션:
    - (a) 화면에 보이는 그대로 표시 (예: "여러 문항 한 번에 비교하기")
    - (b) 비교 대상 문항들의 공통 카테고리/그룹명을 표시 (예: "조직문화 만족도 비교")
    - (c) 사용자가 캡처 시점에 직접 입력
  - 또한 `Q. 질문 원문`(`.result-question-full`)이 비어 있는 케이스의 처리 룰도 정의 필요.
  - 향후 추가될 수 있는 다른 특수 케이스(교차 비교, 그룹 비교 등)에 대한 일반 규칙도 함께 정의.

- [ ] **파일명 규칙**
  - 단일 문항 캡처: `{문항 제목}.png` (정의됨)
  - 다음 케이스의 파일명 규칙은 미정:
    - `targetScaleCompareMode` 상태 (여러 문항 비교) — 대표 카테고리명? 다중 문항 제목 결합? 사용자 입력?
    - 동일 문항을 같은 세션에서 여러 번 추출하는 경우 (덮어쓰기 / 자동 넘버링 `_2`, `_3`)
    - 파일명에 사용 불가능한 문자(`/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|`)가 포함된 문항 제목의 sanitize 룰

- [ ] 여러 문항 한 번에 비교하기(`targetScaleCompareMode`) 상태에서의 캡처 대상 단위 정의 (위 2개 항목과 함께 결정)

- [ ] **전체 문항 일괄 추출 기능** — 각 문항마다 그래프 유형, 정렬 옵션, 데이터 테이블 펼침 등 사용자가 설정한 상태가 다르므로, 일괄 추출 시 각 문항의 현재 화면 상태 그대로 추출할지 / 별도의 일괄 옵션을 지정할지 / 결과물을 ZIP으로 묶을지 등의 처리 방식 정의 필요

- [ ] 데이터 테이블 숨김 처리 후에도 콘텐츠가 A4 한 페이지에 들어가지 않는 경우의 fallback 처리 방식
