# 설문조사 분석 서비스 화면 정의서

- 문서명: 설문조사 분석 서비스 화면 정의서
- 문서 목적: 홈 화면과 설문조사 분석 대시보드 화면의 UI 구성, 컴포넌트, 설명, 인터렉션을 통합 정의
- 파일 구성: 화면은 `home.html`, `dashboard.html`로 분리되어 있으며, 분석 시작 시 페이지 단위로 이동한다.
- 디자인 톤: 미니멀 흑백 기반 (검정 텍스트, 흰색 배경, 연회색 보조), 둥근 모서리와 넉넉한 여백 중심.

## 1. 홈 화면

- 화면명: 홈 화면
- 파일: `home.html`
- 화면 목적: 사용자가 설문조사 제목을 입력하고 분석에 필요한 파일을 업로드한 뒤, 분석을 시작하는 진입 화면

### 1.2 주요 인터렉션 정의

| 항목 | 설명 |
|---|---|
| 제목 입력 검증 | 제목 입력값이 비어 있으면 분석 시작 버튼은 비활성화 상태 유지 |
| 파일 업로드 방식 | 각 업로드 박스는 `파일 선택` 버튼 클릭과 드래그 앤 드롭을 모두 지원 |
| 파일 형식 검증 | `.csv`, `.xlsx`만 허용하며, 허용되지 않는 형식은 에러 메시지 노출 |
| 업로드 완료 처리 | 파일 업로드 성공 시 솔리드 라운드 박스 + `업로드 완료!` + 체크 아이콘 + 파일명 표시 |
| 파일 재선택 | 업로드 완료 상태에서도 박스 하단 `다시 선택하기` 액션으로 파일을 교체할 수 있음 |
| 파일 형식 검증 | 확장자 검증을 통과한 파일은 내용 파싱 후 아래 기준을 만족해야 업로드 완료로 전환됨 |
| 문항 코드북 스키마 검증 | 필수 컬럼 `question_no`, `question_label`, `response_type`, `data_column_role`을 모두 포함해야 함 |
| 응답 데이터셋 공통 검증 | 필수 컬럼 `survey_year`, `respondent_no`를 포함해야 함 |
| 숫자형/라벨형 구분 검증 | 응답 컬럼 셀의 값 패턴을 샘플링해 숫자형 슬롯에는 숫자 코드, 라벨형 슬롯에는 텍스트 라벨이 우세해야 함. 반대 유형 감지 시 에러로 처리 |
| 분석 시작 버튼 활성화 | 제목 입력 완료 + 코드북 업로드 + 숫자형 업로드 + 라벨형 업로드 완료 시 활성화 |
| 분석 시작 버튼 클릭 | 필수값 누락 시 각 에러 메시지 노출, 유효하면 현재 설문조사를 `저장된 대시보드`에 저장 후 `dashboard.html` 페이지로 이동 |
| 설문조사 저장 | 분석 시작 시 제목, 파일명, 파일 내용, 생성 일시를 브라우저 저장소에 저장해 이후 재업로드 없이 다시 열 수 있도록 함 |
| 저장된 대시보드 리스트 | 상단 `저장된 대시보드 리스트` 버튼 클릭 시 저장된 설문조사 목록 모달 오픈. 항목 클릭 시 해당 설문의 대시보드로 이동, 삭제 버튼으로 개별 삭제 가능 |
| 가이드 링크 이동 | 가이드 배너 클릭 시 파일 준비 가이드 화면 또는 외부 문서로 이동 |

### 1.3 화면 메모

- 홈 화면은 분석을 시작하기 위한 온보딩 성격의 화면이다.
- 사용자가 무엇을 준비해야 하는지 직관적으로 이해할 수 있도록 업로드 항목명을 명확히 구분한다 (`숫자형` / `라벨형` 표기).
- 세 개의 업로드 파일은 역할이 다르므로, 각 파일의 예시명 또는 포맷 설명이 추가되면 사용성이 높아질 수 있다.
- 업로드 후에도 파일을 교체할 수 있음을 UI로 명시해 사용자가 잘못 업로드했을 때 쉽게 다시 선택할 수 있도록 한다.
- 섹션 제목과 업로드 라벨, 드롭존 안내문과 `파일 선택` 버튼 사이는 충분한 상하 여백을 두어 시각적 위계를 확보한다.
- 한 번 업로드한 설문조사는 브라우저 저장소에 자동 저장되어 재방문 시 리스트에서 바로 열 수 있다. 저장소 용량 한계를 넘는 경우 저장을 건너뛰고 사용자에게 안내한다.

### 1.4 파일 형식 검증 기준

| 업로드 슬롯 | 필수 컬럼 | 내용 검증 | 오류 처리 |
|---|---|---|---|

| 응답 데이터셋 _ 숫자형 | `survey_year`, `respondent_no` | 식별자 컬럼 이후 응답 컬럼을 샘플링한 결과 숫자 값 비율이 높아야 함 (라벨 텍스트 우세 시 라벨형 파일로 판단) | `라벨형 파일로 보입니다. 숫자형 파일을 업로드해 주세요.` 등 안내 노출 |

## 2. 설문조사 분석 대시보드 화면

- 화면명: 설문조사 분석 대시보드
- 파일: `dashboard.html`
- 화면 목적: 문항 리스트에서 분석 대상 문항과 분석 기준 문항을 선택하고, 필터를 적용해 설문 데이터를 시각화 및 비교 분석하는 화면

| Footer | 저작권 표기 | Text | 하단 푸터에 저작권 정보 표시. 예: `(주)퍼시스 Copyright 2026 fursys Inc.` | 없음 |

### 2.2 주요 인터렉션 정의

| 항목 | 설명 |
|---|---|
| 문항 선택 | 좌측 문항 리스트에서 문항을 드래그하여 `분석 대상 문항` 또는 `분석 기준 문항` 영역에 드롭해 선택 |
| 분석 대상 선택 제한 | 분석 대상 문항은 최대 10개까지 선택 가능 |
| 분석 기준 선택 제한 | 분석 기준 문항은 최대 2개까지 선택 가능 |
| 필터 적용 방식 | 필터는 전체 모집단을 제한하는 역할이며, 적용 시 전체 N값이 감소 |
| 분석 기준 적용 방식 | 분석 기준은 데이터를 그룹으로 나누는 역할이며, N값 자체는 유지하고 결과만 분리 표시 |
| 기준 2개 선택 시 | 그래프 영역을 2개 컬럼 또는 다중 패널 형태로 분할하여 비교 표시 |
| 문항 제거 | 선택된 문항 태그의 삭제 버튼 클릭 시 해당 문항 선택 해제 |
| 저장 | 현재 선택 문항, 기준 문항, 필터, 화면 상태를 저장 |
| 신규 분석 | `새 설문조사 분석하기` 클릭 시 기존 화면과 분리된 신규 분석 세션 시작 |

### 2.3 화면 메모

- 좌측 문항 패널은 카테고리 중심으로 문항을 탐색하고, 드래그 앤 드롭으로 우측 분석영역과 연결된다.
- 필터는 전체 모집단을 줄이는 기능이고, 분석 기준은 비교 축을 나누는 기능이므로 역할을 분리해 표시해야 한다.
- 응답 형식에 따라 적합한 차트 타입을 자동 매핑할 수 있도록 별도 기준 문서가 필요하다.

## 3. 추가 정의 필요 사항

- 홈 화면의 가이드 링크 연결 대상 확정
- 홈 화면에서 업로드 파일별 예시 포맷 또는 샘플 다운로드 제공 여부
- 문항 리스트의 카테고리 체계 및 명칭 규칙
- 분석 기준으로 사용 가능한 문항과 사용 불가능한 문항의 판별 기준
- 응답 형식별 차트 매핑 기준 문서화
- 데이터 테이블 노출 위치와 다운로드 제공 여부
- 저장하기 시 저장 범위 확정
  - 필터 저장 여부
  - 문항 조합 저장 여부
  - 차트 레이아웃 저장 여부
