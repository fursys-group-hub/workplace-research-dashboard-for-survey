# 대시보드 화면 정의서

## 1. 화면 개요
- 화면명: 설문조사 분석 대시보드
- 파일명: `dashboard.html`
- 목적: 업로드한 코드북 기반으로 문항을 탐색하고, 분석 대상/기준 문항을 선택한 뒤 필터와 결과 영역을 확인한다.

## 2. 주요 사용자 목표
- 현재 설문 제목을 확인한다.
- 저장된 대시보드 리스트를 조회한다.
- 현재 대시보드를 저장한다.
- 새 대시보드를 만든다.
- 좌측 문항 패널에서 문항을 탐색/검색/확장한다.
- 문항을 드래그해 분석 영역에 배치한다.
- 필터를 적용해 표본수 변화를 확인한다.

## 3. 화면 구성

### 3.1 상단 헤더
- 좌측
  - `purple6studio` 로고
  - 현재 설문 제목
- 우측 액션
  - `현재 대시보드 저장하기`
  - 세로 구분선
  - `저장된 대시보드 리스트`
  - `새 대시보드 만들기`

### 3.2 헤더 UX 원칙
- 설문 제목은 로고보다 더 큰 1차 정보다.
- 저장 버튼은 주요 액션이므로 검은색 배경을 사용한다.
- 리스트 버튼은 홈 화면과 동일한 pill 계열 UI를 사용한다.
- 로고는 클릭해도 이동하지 않는다.

## 4. 좌측 문항 패널

### 4.1 기본 구조
- 제목: `문항 리스트`
- 검색창: 패널 최상단 고정
- 문항 렌더링 영역: 패널 내부 독립 스크롤

### 4.2 코드북 기반 렌더링
- 데이터 소스
  - 홈 화면에서 저장된 코드북 파일
- 매핑 규칙
  - `category_1` → 1차 아코디언
  - `category_2` → 2차 아코디언
  - `question_label` → 드래그 가능한 문항 카드
  - `question_full` → 확장 상태에서 보조 설명으로 활용
- 초기 상태
  - 최초 접속 시 모든 1차·2차 아코디언은 닫혀 있다
  - 사용자가 직접 클릭해야 펼쳐진다
  - 검색을 지우고 다시 빈 상태가 될 때에도 모두 닫힌 상태로 돌아간다
- 화살표 아이콘 위치
  - 1차·2차 아코디언 모두 오른쪽 정렬
  - 접힘: 오른쪽 방향(`>`), 펼침: 아래 방향(`v`)

### 4.3 문항 카드
- 기본 상태
  - `question_label`만 표시
- 확장 상태
  - `question_full` 값이 실제로 있는 문항만 `question_label | Q. question_full` 구조로 표시
  - `question_full` 값이 없으면 라벨만 유지
- 드래그 시 전달 값
  - 항상 `question_label`만 사용

### 4.4 문항 검색
- 위치: 패널 최상단
- 검색 아이콘 포함
- 검색 대상
  - `category_1`
  - `category_2`
  - `question_label`
  - `question_full`
  - `question_no`
- 검색 시
  - 관련 아코디언 자동 확장
  - 결과 없으면 `검색 결과가 없습니다.` 표시

### 4.5 문항 패널 확장
- 조작 요소
  - 패널 오른쪽 경계에 붙은 세로 핸들
- 동작 방식
  - 패널이 메인 영역을 밀지 않고 위로 덮는 오버레이형 확장
- 확장 상태 특징
  - 패널 폭 증가
  - 문항 카드에서 `question_full` 표시 가능
  - 메인 대시보드 영역의 원래 위치와 크기는 유지
  - 핸들은 패널의 일부로 동작하며, 별도 떠 있는 버튼처럼 보이지 않음

### 4.6 패널 스크롤
- 좌측 패널 자체에 독립 스크롤 존재
- 메인 영역 스크롤과 분리
- 제목/검색 영역은 sticky 처리

## 5. 메인 분석 영역

### 5.1 필터 바
- 메인 대시보드 영역 내부 상단 바 형태
- 왼쪽/위/오른쪽에 딱 붙는 직사각형 내부 헤더 느낌
- 스크롤 시 메인 영역 내부에서 `sticky` 유지
- 현재 표시 항목
  - 성별
  - 연령
  - 세대
  - 업종
  - 직무
  - 규모
  - `N = 1,248`
- 현재 동작
  - 필터 적용 수에 따라 샘플 N 값이 감소하는 목업 로직

### 5.2 분석 대상 문항 영역
- 제목: `분석 대상 문항`
- 제한: 최대 10개
- 입력 방식
  - 좌측 문항 카드 드래그 앤 드롭

### 5.3 분석 기준 문항 영역
- 제목: `분석 기준 문항`
- 제한: 최대 2개
- 입력 방식
  - 좌측 문항 카드 드래그 앤 드롭

### 5.4 드래그 결과 칩
- 검은색 pill 형태
- 삭제 버튼 포함
- 표시 텍스트는 `question_label`만 사용

### 5.5 결과 영역
- 샘플 막대 차트
- 샘플 빈도/비율 테이블
- 현재는 목업 데이터 기준 표시

## 6. 저장된 대시보드 리스트 모달
- 대시보드 화면 내 오버레이 모달로 표시
- 홈 화면과 동일한 리스트 패턴 유지
- 기능
  - 목록 조회
  - 제목 클릭 시 해당 대시보드 열기
  - 개별 삭제

## 7. 상태 및 동작 규칙

### 7.1 현재 설문 제목 반영
- `sessionStorage['survey.currentId']`
- `sessionStorage['survey.title']`
- `localStorage['p6s.surveys']`

### 7.2 드롭 제한
- 분석 대상: 최대 10개
- 분석 기준: 최대 2개
- 초과 시 alert 표시

### 7.3 확장 모드
- `page.panel-expanded` 클래스 기반 제어
- 세로 핸들 클릭 시 토글

## 8. 현재 UX 특징
- 헤더 위계 정리 완료
- 좌측 패널은 실제 코드북 구조를 반영하는 상태
- 2단 아코디언 + 검색 + 독립 스크롤 지원
- 확장 시 메인 레이아웃은 유지되고 패널만 덮음
- 필터 영역은 카드형이 아니라 메인 영역 내부 상단 바 형태로 정리됨

## 9. 이번 문서에서 제외한 범위
- 설문별 동적 필터 추천 규칙
- 사용자가 추가하는 필터 규칙
- 저장된 대시보드의 데이터 업데이트 규칙
- 필터/문항 배치 상태까지 포함하는 고급 저장 규칙
